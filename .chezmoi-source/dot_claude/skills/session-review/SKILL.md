# セッション振り返り

ユーザーが任意のタイミングで実行する手動起動の振り返りスキル。

以下の3項目を順に点検し、それぞれ結果を報告する。
該当する利用が無い項目はスキップしてよい。
改善が無い項目は「指摘無し」と明記する。

追記の書き方は1コンセプト1行を基本とし、冗長な説明や自明な情報は避ける（特に常時ロードされる指示文書では簡潔さが重要）。

## 1. プロジェクト固有の学び（CLAUDE.md更新提案）

今セッションで得た、当該プロジェクト固有の知見を反映すべきか検討する。

### 振り返りの観点（CLAUDE.md）

Claudeが今後同種の作業をする際に役立つコンテキストが欠けていなかったか振り返る。

- 使用または発見したbashコマンド
- 守るべきコードスタイル・パターン
- 有効だったテスト手法
- 環境・設定の癖
- 遭遇した警告・落とし穴
- ユーザーから繰り返し同じ修正指示を受けた箇所

一度きりで再発しなさそうな修正や、自明な情報は対象外とする。

### 反映先

`find . -name "CLAUDE.md" -o -name "CLAUDE.local.md"`で候補を洗い出し、以下から選ぶ。

- `CLAUDE.md`: チーム共有（gitコミット対象）
- `CLAUDE.local.md`: 個人・ローカルのみ（gitignore対象）

### 提案と適用

追記候補ごとに反映先・理由（Why）・差分を提示してユーザー承認を得てから`Edit`で適用する。

## 2. pyfltr振り返り

当該セッションでpyfltrを使った場合のみ実施する。
未使用なら本項目はスキップする。

### 振り返りの観点（pyfltr）

- pyfltrの挙動・出力で分かりにくかった点や混乱した点
- 既存ドキュメントの記述漏れ・誤り

### 改善提案先（pyfltr）

実際の反映はユーザーが別途行うため、本スキルでは提案までにとどめる。

- pyfltr本体の挙動・メッセージ
- `plugins/agent-toolkit/skills/pyfltr-usage/SKILL.md`（使い方リファレンス）

## 3. agent-toolkit振り返り

`agent-toolkit`プラグイン（スキル・フック・サブエージェント）および
`agent-toolkit`ルール（`agent.md`・`styles.md`）を対象とした振り返り。

### 振り返りの観点（agent-toolkit）

- `agent.md`の手順（バグ修正の根本原因調査、検証→コミットの順序など）に違反した箇所
- プラグイン・ルールの指示で分かりにくかった点や混乱した点
- 既存指示の記述漏れ・誤り

### 改善提案先（agent-toolkit）

実際の反映はユーザーが別途行うため、本スキルでは提案までにとどめる。

- `agent-toolkit`プラグイン（スキル・フック・サブエージェント）
- `~/.claude/rules/agent-toolkit/`配下のルール
