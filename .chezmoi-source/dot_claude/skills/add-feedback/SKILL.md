# フィードバック投入

## 起動方針

`~/.config/agent-toolkit/feedback-inbox.enabled`が存在しない場合は、
フィードバック蓄積機能が無効である旨を1文示して終了する。

`~/private-notes`が存在しない場合は、手動で`~/private-notes`をクローンしてから
再度実行する旨を1文示して終了する。

## ステップ1: 入力の確認

ユーザーが提示したフィードバック本文を読み取る。
本文が提示されていない場合はユーザーに提供を求める。

## ステップ2: 対象リポジトリの判別

各提案項目の主題から`target_repo`のリポジトリパスを判定する。

- pyfltrリポジトリへの提案は`~/pyfltr`
- agent-toolkitプラグイン・配布ルール・dotfiles配布物への提案は`~/dotfiles`
- 現在作業中のプロジェクトリポジトリへの提案はカレントディレクトリ

判別が困難な項目はユーザーへ確認する。

## ステップ3: メッセージの整形

各メッセージは単体で読んで意味が完結する形に整える。
同一出典から派生する複数提案では、本文へ次の情報を必ず含める。
直近コミット文脈・対話文脈への暗黙参照を含む提案でも同様とする。

- 出典URL・参照元（外部リポジトリ・記事・PR等）
- 関連する派生提案との関係（同一出典・代替案・前提依存など）
- 対象を一意に特定するために必要な前置き

対象ファイル・対象節が特定できる場合はメッセージ冒頭で明示する。
項目間に共通する前置きは各メッセージ本文へ複写する。
フィードバックは個別ファイルへ分割して保管されるためファイル間参照は機能しない。

## ステップ4: 投入

target_repoごとに、対応するメッセージ群を位置引数で渡して`dotfiles-fb add`を呼び出す。

```sh
dotfiles-fb add <repo-path> <message1> [<message2> ...]
```

- `<repo-path>`: target_repoの絶対パスまたは`~`展開可能パス
- `<message>`: 自由本文。改行を含めてよい
- 複数のtarget_repoを投入する場合はリポジトリ別に複数回呼び出す

## ステップ5: 結果の提示

`dotfiles-fb add` CLIの標準出力をそのままユーザーへ提示する。
