# Claude Code 設定管理

本リポジトリのClaude Code関連のファイルは、他のプロジェクトで利用するためのテンプレートも兼ねている。

`claudize` コマンドでプロジェクトごとに必要なファイルを配布・同期する。

配布・同期したファイルは他のユーザーと共有される可能性があるため、`claudize` のような個人的なツールへの依存を含めないようにしている。
配布元の管理やツール固有の情報はこの文書に集約する。

## `claudize` コマンドの使い方

```bash
# プロジェクトディレクトリで実行
cd ~/your-project
claudize
```

## `claudize` の管理対象ファイル

- `.claude/rules/agent.md`: 自動化すべき部分とユーザー確認すべき部分のバランス調整、コード品質の維持のためのルール
- `.claude/rules/{言語}.md`: 言語固有のコーディング規約のうち一般的なもの (該当言語のファイルが存在する場合のみ配布)
- `.claude/rules/{言語}-test.md`: 言語固有のテスト方針のうち一般的なもの (該当言語のファイルが存在する場合のみ配布)
- `.claude/rules/markdown.md`: Markdown記述スタイルの統一のためのルール
- `.claude/rules/rules.md`: ルール作成時のガイドライン
- `.claude/rules/skills.md`: スキル作成時のガイドライン

CLAUDE.md はプロジェクト固有の情報を記述するファイルとして、claudize の管理対象外。
プロジェクトごとに手動で管理する。(`/init`コマンドなどを活用するのも手)

ルールファイルの同期時、既存ファイルの YAML frontmatter（`paths` 等）は維持される。
(プロジェクトによって追加のパターンを記述したい場合など向け)
