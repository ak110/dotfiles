# 次セッションへの引き継ぎ用markdown出力

会話が発散してコンテキストが肥大化したり、要件確認と手戻りが重なって見通しが悪くなった時に、現状の要点を整理したmarkdownを作業プロジェクトルート直下に書き出すスキル。
出力ファイルを添えて新しいClaude Codeセッションを立ち上げ直し、冒頭でそのファイルを読ませることで、これまでの経緯を短く伝えて作業を再開できる。

セッション自体を履歴として残したい場合は`export-session`スキル（`claude-session-export` CLIで会話全文をmarkdown化）を使う。
本スキルは要点の抽出・整理に特化しており、用途が直交する。

## 用途と前提

- 想定場面:
  - 要件対話や計画ファイル作成中に議論が発散し、何が決まって何が未決かの把握が困難になってきた時
  - コンテキストが肥大化して応答品質や速度が低下してきた時
  - セッション終了直前に、現時点のスナップショットだけ残して後日再開したい時
- 前提: ユーザーが明示的にこのスキルを呼んでいる状態。実行中のコード変更やコミット操作は行わず、ファイル出力のみに専念する

## 出力先とファイル名

- 出力先: 作業プロジェクトのルート直下
- ファイル名: `resume-<会話テーマを要約した英語文>.md`（会話の主題を短い英語のケバブケースで要約する。例: `resume-agent-toolkit-cleanup.md`）
- ユーザーがgit管理外として扱うか任意の場所へ移動するかは本スキル側で決め打ちしない。ユーザー判断に委ねる

同一テーマで再呼び出しされた場合は既存ファイルの上書きでよい（更新日時はファイルシステム側で追跡できるため本文にタイムスタンプは含めない）。

## 出力テンプレート

以下のテンプレートに沿ってmarkdownを生成する。
情報がない節は「（該当なし）」と明記して空節にしない。

```markdown
# セッション引き継ぎメモ

## 発端の指示

<ユーザーの初期要望・最初のトリガーとなった依頼内容を原文ベースで>

## 決定事項

- <ユーザーとの対話で合意済みの方針・仕様・制約>
- ...

## 未決事項

- <合意に至っていない論点・ユーザー確認待ちの質問>
- ...

## 現在の進捗

- <着手済みタスクとその状態>
- <未着手タスク>

## 関連ファイル・パス

- 計画ファイル: <`~/.claude/plans/*.md`のパスがあれば>
- 編集対象ファイル: <絶対パスまたはリポジトリ相対パス>
- 参考ドキュメント: <参照したドキュメントのパス>

## これまでの試行錯誤の要点

- <検討したが採用しなかった方針・棄却理由の要約>
- <ハマった箇所と解消経緯>
```

## 出力後のユーザー案内

ファイル生成後、以下の要点を口頭でユーザーに伝える（ファイル内には書かない）。

- 出力ファイルの絶対パス
- 新しいClaude Codeセッションを立ち上げ直し、冒頭でこのファイルを読ませてから作業を再開する運用を推奨する旨
- 本ファイルはgit管理外として扱うか、任意の場所へ移動するかをユーザーが判断できる旨

## 注意点

- 決定事項と未決事項を厳密に分ける。推測や前提の補完で未決事項を補完しない（次セッションでの誤解を招くため）
- ユーザーの発話・確認済み合意のみを「決定事項」に書く。エージェントの推測は書かないか、明示的に「推測」と注記する
- 過去の試行錯誤も簡潔に残す。同じ論点を再検討して同じ結論に至る手戻りを防ぐため
- コード断片や詳細仕様は極力含めず、参照先のファイルパスで代替する。引き継ぎの目的はコンテキスト削減のため
- 本スキル実行中に推測判断が必要な箇所があれば、先にユーザーに確認して確定させる（スキル本体の趣旨に反するため）
