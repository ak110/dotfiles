# spec-driven昇格ワークフロー

次版ドキュメントを現行版ドキュメントへ統合し、`docs/v{next}/`を削除する。

## 手順

1. `spec-driven`スキルを呼び出し、用語・配置規約・テンプレートを確認する
2. 昇格対象の`docs/v{next}/`を確認する
3. 各次版ドキュメントの「現行版ドキュメントの対応先」を確認する
4. 次版ドキュメントの内容を`docs/features/`または`docs/topics/`へ統合する
5. コード中の参照コメントを現行版ドキュメントのパスへ書き換える
6. `docs/v{next}/`を削除する
7. `docs/v{next}/`への参照が残っていないことを確認する
8. リンクと参照コメントの指し先を検証する

## 統合方針

- 新規機能は現行版ドキュメントとして作成する
- 既存機能の改修は対応先の現行版ドキュメントへ統合する
- 横断ドキュメントは`docs/topics/`へ統合する
- 主要設計判断は現行の判断として整理する
- 却下した代替案は判断履歴として残す
