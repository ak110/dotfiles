# テンプレート集

spec-drivenスキルで使う恒久ドキュメント（機能・横断・`README.md`）・作業テーマ`.md`・一時ドキュメント・参照コメントの書式テンプレート。
参照コメントの配置方針などはSKILL.md本体を参照する（本ファイルは書式のみ提示する）。

恒常配置（`docs/features/`・`docs/topics/`）の最終形と開発中配置（`docs/v{next}/`配下）の作業版は同一骨格のテンプレートを基にする。
昇格（`spec-driven-promote`スキル）時にマージ統合するため、節構成を揃えておく。
本ファイル中の具体パス表記は既定値であり、プロジェクトのドキュメントで別の配置が規定されている場合はそちらに読み替える（`spec-driven`スキルSKILL.mdの「配置の既定と上書き」節を参照）。

## 横断ドキュメント（`{トピック名}.md`）

複数機能で参照される設計判断・ガイドライン・共通用語をまとめる。
機能ドキュメントのテンプレートを流用し、以下の差分だけ意識する。

- 受け入れ基準は機能単位でないため省略してよい
- 「スコープ」は対象範囲（どの機能から参照されるか・どういう場面で参照するか）に読み替える
- 「背景」には横断化した経緯（どの機能間で共通と判断したか）を書くと再利用しやすい
- 開発中配置の横断ドキュメントでも「恒常配置の対応先」節を設け、昇格時のマージ対象（`docs/topics/{トピック名}.md`）を明示する

## バージョン概要（`README.md`）

`docs/v{next}/README.md`に置く次期バージョン全体の入口。
次期リリース作業期間中のみ存在し、昇格時に役目を終える（開発中バージョンディレクトリごと削除される）。
ステータス列は含めない（進行状況の管理は`.working.md`・TaskCreateに委ね、恒久ドキュメントに陳腐化する情報を載せないため）。

````markdown
# v{next}

## バージョン目的

（このバージョンで実現したい目的・テーマ）

## 作業テーマ一覧

（本バージョンで追加・変更する作業テーマ。各ドキュメントへのリンクと一行説明。昇格時のマージ対象機能も併記すると全体像を把握しやすい）

- [docs/v{next}/{作業テーマ名1}.md]({作業テーマ名1}.md) — 一行説明（対応先: `docs/features/{機能名}.md`）
- [docs/v{next}/{作業テーマ名2}.md]({作業テーマ名2}.md) — 一行説明（新規機能）

## 横断ドキュメント一覧

（複数機能で参照される設計判断・ガイドライン・共通用語）

- [docs/v{next}/{トピック名1}.md]({トピック名1}.md) — 一行説明
- [docs/v{next}/{トピック名2}.md]({トピック名2}.md) — 一行説明
````

## spec-designer呼び出し

ステップ2後半（`ExitPlanMode`直後）にメインが呼び出す設計ドキュメント立ち上げエージェントの呼び出しテンプレート。
計画ファイルとリサーチ結果ファイル群を入力に、開発中`.md`骨子・`.working.md`初版・`README.md`エントリ・必要なら横断ドキュメントを一括生成する。
実装コードは編集しない（`spec-implementer`の責務）。

```text
以下の作業テーマの設計ドキュメントを立ち上げてください。

作業テーマ名: {作業テーマ名}
次期バージョン: {next}
区分: 新規追加 / 既存機能改修
計画ファイル: `~/.claude/plans/{自動生成ファイル名}.md`
リサーチ結果ファイル: {`docs/v{next}/{作業テーマ名}.research-{nn}.md` の一覧}
恒常配置の対応先候補: `docs/features/{機能名}.md` または `docs/topics/{トピック名}.md`（既存改修時、新規は「該当なし」）

出力:
- `docs/v{next}/{作業テーマ名}.md` 骨子（本ファイルの作業テーマテンプレートに準拠）
- `docs/v{next}/{作業テーマ名}.working.md` 初版（調査メモ・判断経緯・タスク分解草案・既存改修時は改修前スナップショットを含む）
- `docs/v{next}/README.md` の作業テーマ一覧にエントリ追加
- 必要なら横断ドキュメント `docs/v{next}/{トピック名}.md` を新規作成し、README.mdの横断ドキュメント一覧にも追加

戻り値: 生成／更新したファイルパス一覧と1行サマリー、判定が曖昧だった事項のエスカレーション
制約: 実装コードは編集しない。書き込み対象は `docs/v{next}/` 配下に限定する。TaskCreate登録は行わない（メインの進行管理権）
```

## レビュアー呼び出し（spec-reviewer・code-quality-reviewer）

ステップ3で機能単位に直列起動するレビュアーの呼び出しテンプレート。
両レビュアーに共通する引数は「作業テーマ名・開発中`.md`のパス・`BASE_SHA`・レビュー対象外の一時ファイル一覧・出力ファイルパス」の5点。
`spec-reviewer`には加えてドキュメント整合性検査のため差分に含まれる他の恒久ドキュメントと、既存改修時は改修前の該当節スナップショット、そして計画ファイル（転記漏れ検査用）を渡す。

`BASE_SHA`はステップ2で記録した`git rev-parse HEAD`の結果を使う。
`spec-implementer`はタスク単位でコミットしない設計のため、レビュー時点の差分は未コミットの作業ツリー差分として扱う。
差分取得は`git diff {BASE_SHA}`と未追跡ファイルを含む作業ツリー参照（`git status`・`Read`）で行う。
一時ファイル（`.working.md`・`.research-*.md`・`.review-spec.md`・`.review-quality.md`）は実装完了時に削除される想定のため、評価対象外として除外する。

指摘は呼び出し元指定の出力ファイルへ書き出し、戻り値は判定・件数のみに絞る。
差し戻しループでは同一ファイルを上書きする（前回指摘との混在を避けるため）。

### spec-reviewer

```text
以下の作業テーマについて仕様適合性・ドキュメント整合性レビューを実施してください。

作業テーマ名: {作業テーマ名}
開発中ドキュメント: `docs/v{next}/{作業テーマ名}.md`
差分内の他の恒久ドキュメント: {同一開発中ディレクトリ配下の他作業テーマ・横断・README.mdのうち差分に含まれるもの}
恒常配置側の該当ドキュメント: `docs/features/{機能名}.md` または `docs/topics/{トピック名}.md`（既存改修時のみ）
改修前の該当節スナップショット: `docs/v{next}/{作業テーマ名}.working.md`内の該当節（既存改修時のみ）
計画ファイル: `~/.claude/plans/{自動生成ファイル名}.md`
BASE_SHA: {ステップ2で記録したHEAD}
対象外ファイル: `docs/v{next}/{作業テーマ名}.working.md`・`docs/v{next}/{作業テーマ名}.research-*.md`・`docs/v{next}/{作業テーマ名}.review-spec.md`・`docs/v{next}/{作業テーマ名}.review-quality.md`
出力ファイル: `docs/v{next}/{作業テーマ名}.review-spec.md`（差し戻しループでは上書き）

差分取得: `git diff {BASE_SHA}`と、未追跡ファイルを含む作業ツリー参照で行うこと
制約: 書き込みは出力ファイル1つのみ。実装者レポートは鵜呑みにせず、コードと差分で独立検証すること
戻り値: 判定（✅/❌）と指摘件数
```

### code-quality-reviewer

```text
以下の作業テーマについてコード品質レビューを実施してください。

作業テーマ名: {作業テーマ名}
開発中ドキュメント: `docs/v{next}/{作業テーマ名}.md`
BASE_SHA: {ステップ2で記録したHEAD}
対象外ファイル: `docs/v{next}/{作業テーマ名}.working.md`・`docs/v{next}/{作業テーマ名}.research-*.md`・`docs/v{next}/{作業テーマ名}.review-spec.md`・`docs/v{next}/{作業テーマ名}.review-quality.md`
出力ファイル: `docs/v{next}/{作業テーマ名}.review-quality.md`（差し戻しループでは上書き）

差分取得: `git diff {BASE_SHA}`と、未追跡ファイルを含む作業ツリー参照で行うこと
制約: 書き込みは出力ファイル1つのみ。coding-standardsスキルを事前に呼び出し、品質基準に従うこと
戻り値: Assessment（approve/reject）と指摘件数（Critical/Important/Minor別）
```
