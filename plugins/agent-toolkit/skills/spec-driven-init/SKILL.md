# spec-driven導入ワークフロー

既存プロジェクトに`spec-driven`を導入するための前段階として、恒常配置（`docs/features/`・`docs/topics/`）の初版を整備する。
実行後は通常の`spec-driven`ワークフローを開始できる状態（恒常配置の整備が済み、開発中バージョンディレクトリは未作成）となる。

起動時にまず`Skill`ツールで`spec-driven`を呼び出し、用語・配置規約・テンプレートを読み込んでから本スキルの手順を実行する。
本スキル本文は初回整備に固有の差分のみを記述する。

## 適用条件と非適用条件

`spec-driven`の「適用条件と非適用条件」に加え、本スキル固有の条件。

適用対象:

- 既存プロジェクトでspec-drivenをこれから導入したい
- 既存の`docs/v*/`群が残っており、恒常配置へ統合したい
- 既存のソースコードやドキュメントから機能・横断トピックの単位を整理し直したい

非適用:

- 恒常配置（`docs/features/`・`docs/topics/`）が既に整備されている（通常の`spec-driven`で進める）
- リリース完了後の昇格作業（`spec-driven-promote`を使う）
- ドキュメントが全く無いプロジェクト（通常の`spec-driven`ワークフローで新規作成する方が効率的）

## 手順

1. プロジェクトの現状を走査する
   - ソースコード: 主要モジュール・エントリーポイント・公開インターフェースを列挙する
   - ドキュメント: `README.md`・`docs/`配下・`CLAUDE.md`などから既存の機能記述を抽出する
   - 既存`docs/v*/`群: 存在する場合は各バージョン配下のファイルを確認し、最新内容を把握する
2. 機能単位と横断トピック単位の候補をユーザーに提示する
   - 機能単位: 利用者視点でまとまった機能（例: 認証・通知・検索・請求）
   - 横断トピック単位: 複数機能で参照される設計判断・ガイドライン・共通用語
    （例: エラーハンドリング方針・命名規則・共通データモデル）
   - 区分が曖昧なものは機能として扱い、後から横断へ昇格する段階的運用を推奨する
3. ユーザーとの対話で単位を確定する。迷うファイルは独自判断で進めずユーザーに判断を仰ぐ
4. 確定した単位ごとに`docs/features/{機能名}.md`または`docs/topics/{トピック名}.md`を作成する
   - テンプレートは`spec-driven`スキルの`references/templates.md`を流用する
   - 既存ドキュメントから背景・スコープ・主要設計判断・却下した代替案を抽出して記入する
   - 既存`docs/v*/`群の内容は対応する恒常配置へマージする。
     バージョン横断で重複する場合は最新版を優先し、過去バージョンでしか言及されていない「却下した代替案」は
     恒常配置の該当節に追記する（過去の判断履歴を失わないため）
5. 実装コード中の参照コメントを一括書き換えする
   - 旧: `docs/v{version}/{機能名}.md`等
   - 新: 手順3で確定した恒常配置パス
   - リポジトリ全体を対象とし、書き換え漏れを残さない（手順6の前に必ず完了させる）
6. 既存`docs/v*/`群を削除する。マージ済みかユーザーに確認してから削除する
7. `.gitignore`へ`docs/v*/.cache/`を追記する。同等パターンが既に存在するならスキップ。
   通常の`spec-driven`ワークフローが生成する中間成果物（`docs/v{next}/.cache/`配下）をgit管理から外すために必要
8. 整合性を検証する
   - 恒常配置の`.md`内リンクが壊れていないか
   - `docs/v*/`への参照がコード・ドキュメント・`README.md`等に残っていないか
   - 参照コメントの指し先が実在するか
9. 完了と検証結果をユーザーに報告する

## 制約

- 引数無しで起動する。走査対象・単位確定・マージ方針はユーザーとの対話で決める
- 既存`docs/v*/`群の削除は破壊的操作のため、マージ完了と参照コメント書き換えをユーザーに明示してから実施する
- 参照コメントの書き換え漏れはバージョンディレクトリ削除後のリンク切れに直結するため、手順5と検証手順8は必ず実施する
- 対話型作業のためセッションを跨ぐ場合は、走査結果と確定済み単位を`~/.claude/plans/`配下の一時ファイルに記録することを推奨する
