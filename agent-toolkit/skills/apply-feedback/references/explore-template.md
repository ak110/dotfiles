# Explore委譲雛形集

`agent-toolkit:apply-feedback`スキル「ステップ2.5: 採否判定前の網羅調査」節から
Exploreサブエージェントへ並列委譲する際の起動プロンプトを集約する。
1フィードバックごとに1つの雛形インスタンスを生成して同一メッセージ内で並列起動する。

## Explore委譲雛形

`Explore`サブエージェントへの起動プロンプトは次のとおり。

    目的: フィードバック{ファイル識別子}の追記先調査。

    対象フィードバック原文:
    {フィードバック原文の転記（`text`コードブロック内）}

    調査観点（全項目を実施し結果を報告）:
    1. 追記先候補ファイルの該当節をReadで確認し既存記述カバー有無を確認する
    2. 同ドメインの既存規範を`grep -rn '<対象キーワード>' agent-toolkit/rules/ agent-toolkit/skills/`で網羅確認する
    3. 関連スキル・関連ルールファイルに重複・矛盾する規範が無いかを確認する
    4. 追記先候補が配置先スキルの管轄範囲に該当するかを`AGENTS.md`・`CLAUDE.md`で確認する
       （該当時は当該スキルの起動要否をメイン側で判断できるよう調査結果へ明記）
    5. 追記後最終形の`wc -l`実測値を算出する（近似不可、実測必須）
    6. 原文の主旨を原文転記から再確認し、解釈誤りが無いかを確認する
    7. フィードバック原文が言及する具体値・識別子の実在確認を実施する。
       対象は具体値（行数閾値・節名の具体値・引用文言など）と
       識別子（バッククォート囲み識別子・スラッシュ区切りサブエージェント名・スキル名・
       定数名・関数名・変数名）とする。
       具体値は`grep -n '<具体値>' <反映先候補ファイル>`で網羅照合する。
       識別子は`grep -rn '<識別子>' agent-toolkit/ pytools/ scripts/ .chezmoi-source/`で網羅照合する。
       実在しない具体値・識別子は「誤記候補」として調査結果へ列挙する。
       誤記が確認された場合は原文主旨を実在するSSOT定義へ読み替えてから採否判定する
    8. 置換系の変更を計画する場合、置換対象箇所（変数名・要素構造・
       イベントハンドラー名等）の実ファイル現物をReadで確認し、置換前文面
       （[現行]ブロック）が実ファイルと一字一句一致する形で報告する
    9. 追記先候補ファイルのスコープ宣言（目的記述・SSOT宣言・冒頭導入文の対象範囲）をReadで確認し、
       追記内容の主題が同スコープ内かを検証する

    報告形式:
    - 各観点の調査結果を項目別に簡潔に列挙
    - 該当行の抜粋を含める
    - 採否判定の材料となる事実のみ列挙し、判定・改善案の提案はしない
    - 担当範囲区分: 全項目担当（未実施観点がある場合は出力冒頭で欠落事実を明示報告する）

    ## 制約

    - `Explore`は規範非読込型サブエージェントのため、必要な規範は本プロンプトへ引用転記済み
    - 改善案・修正案の提案は返却しない（事実列挙のみ）
