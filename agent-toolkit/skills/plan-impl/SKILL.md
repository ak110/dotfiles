# 計画後の計画実行

`ExitPlanMode`後にコンテキストがクリアされた状態で呼び出される可能性がある。計画ファイル自立性を前提とする。
計画ファイルの`レビュー方式`項目を読み取り、`レビュー無し`／`レビュー有り`に分岐する。

1. タスク分解
2. 実装
3. 検証
4. コミット
5. レビュー（`レビュー有り`の場合のみ。`agent-toolkit:careful-review`スキルへ引き継ぐ）

## 1. タスク分解

計画ファイルを起点にタスク分解し、`TaskCreate`で登録する（検証・コミット・レビュータスクも含む）。
過度な細分化は避ける。ファイル削除（`rm`等）はサブエージェントへ委譲しない。
レビュータスクは観点ごとに別サブエージェントを起動する（実装の経緯を持ち込まないため）。

## 2. 実装

分解したタスクを実装する。
下記「実装サブエージェント委譲（plan-implementer）の判断指針」に従い、
メイン直接実装または`plan-implementer`への委譲を選ぶ。

### 実装サブエージェント委譲（plan-implementer）の判断指針

タスクごとに次の手順で委譲可否を決める。

1. タスクの難易度から必要なモデル（opus / sonnet / haiku）を見積もる
2. 必要モデルがメインモデルと同等以上ならメイン直接実装を選ぶ（品質担保のため）
3. 必要モデルがメインモデルより低位なら`plan-implementer`へ委譲する（`model`引数で当該モデル指定）
4. 委譲タスクのうち、編集対象ファイルが独立する2件以上は`run_in_background=true`で並列起動する

難易度を見積もれないタスクはメイン直接実装を選ぶ。
対象ファイル数が多い計画でも、各ファイルの編集が独立かつ必要モデルがメインモデル未満なら手順4に従う。

### サブエージェント起動プロンプト

```text
以下のタスクを{実装 | 修正再実装}してください。

- 計画ファイル: `~/.claude/plans/{自動生成ファイル名}.md`

## タスク

{タスクの具体的な記述}

## 修正指摘（修正再実装時のみ）

{レビュアー指摘テキストとメインが具体化した修正方針（対象ファイルパス・該当位置・修正方針）を
 まとめて記述する。具体化方針は欄内に統合し欄外に分散させない}
```

## 3. 検証

全実装タスク完了後に検証する（実装タスクへの統合も可）。
検証コマンドは計画ファイルの「検証手順」の対象範囲全体。

## 4. コミット

計画ファイルの「コミット方針」に従ってコミットする。

## 5. レビュー

`agent-toolkit:careful-review`スキルへ引き継ぐ。
対象範囲は計画全体（全コミット完了後の差分）、計画ファイルパスは本計画のものを渡す。
不変参照点（`origin/<base-branch>`相当または計画着手前SHA）と一時ファイル一覧を併せて渡す。
以降の起動・指摘統合・修正再実装・amend統合・followup再レビューは`agent-toolkit:careful-review`の手順に従う。
