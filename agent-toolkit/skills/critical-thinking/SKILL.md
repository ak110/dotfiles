# 批判的検討の汎用観点

提案・自身の判断・サブエージェント出力の妥当性を評価する場面で共通基準として用いる。
規範文書改訂・フィードバック判定・レビュー指摘の取捨など、判断を伴う全工程に適用する。

- 既存記述との重複・矛盾: 追記先の既存記述で既にカバーされていないかを確認する。
  既存記述の補強で対処できる場合は新規追加を避ける
- コンテキストコスト対効果: 追記による恒常的なコンテキスト消費に見合う再発防止効果があるかを評価する。
  単発・偶発の事象や、レビューで容易に是正できる事象は除外する
- 適用範囲との整合: 反映先の規範文書が対象とする適用範囲と提案文の抽象化粒度が整合するかを確認する。
  クロスプロジェクト規範へ言語固有・フレームワーク固有事項を持ち込まない
- 論理整合性: 提案文・自身の文面案に含まれる条件式・規定の論理矛盾を採用前に解消する。
  原文転記のままで後続レビューが矛盾を指摘し修正再実装ループを招く構造を検出して整合させる
  - 規範文書を新規執筆・改訂する場合、当該指示が運用される他規範文書
   （フォーマット規定・テンプレート・記号固定の規定など）と整合するかを採用前に照合する
  - 並列列挙された根拠・理由・条件（`AかつB`・`Aの禁止・Bの限定`等の複合構造）は
    各要素を分解し個別に実態整合を検証する。複合文全体としての整合性確認では各要素の検証漏れが生じるため
- 実例運用との照合: 外部レビュー（codexレビュー・自動lint・サブエージェント）が
  プロジェクトの規範・スタイル・命名規則・運用手順への抵触を主張することがある。
  採否を確定する前に直近コミット履歴・既存ファイル・既存設定値で実例運用と照合する。
  実例運用と乖離する指摘は規範側の暗黙慣習化を疑い、規範文書の明文化または当該指摘の不採用を選択する
