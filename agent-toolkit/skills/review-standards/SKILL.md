# レビュー品質

## レビューの基本姿勢

- 機械チェック（lint・format・textlint・型検査など）で判定できる項目は扱わない。
  補助スクリプトの実行もレビュー時には行わない
- 確信度の低い指摘は記載しない。
  対象差分・周辺コード・参照仕様を確認したうえで、確実な誤り・規約違反・ロジック欠陥と判断できる項目のみ指摘する
- nice-to-have・主観的提案・スタイル懸念は指摘対象外とする。
  プロジェクト方針やルールに明示されていない好みの選好は指摘しない
- 既知の事前存在問題、明示された修正対象外の領域、計画範囲外で別途対応する旨が示された箇所は指摘対象外とする
- 初回レビューでは対象範囲の指摘をすべて列挙する。
  followupレビューでは前回指摘の対応確認のみを行い新規指摘は追加しない（明らかな回帰のみ例外）
- 実装担当者の報告は参考情報として扱い、実装事実はコードと差分で確認する

## 指摘の表現

- 指摘は必ず`path:123`形式の根拠を含める。
  対象が複数行にわたる場合は代表行または範囲のいずれかを示す
- 推測で断定せず、確認した事実に基づいて記述する。
  確認できない事項は指摘せず、必要なら確認手段を取った上で再評価する
- 悪い例の文面や機械チェック辞書の検出語を指摘文へ転記しない。
  該当する観点を抽象的に説明する形で記述する（指摘文経由で対象コンテキストへ違反パターンが混入する事態を避ける）
