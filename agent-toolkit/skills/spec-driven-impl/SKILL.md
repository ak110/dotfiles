# 自律実行による次版ドキュメント実装

`agent-toolkit:spec-driven`で作成した次版ドキュメントを、ユーザーが応答できない前提で実装まで進める。
確認すべき事項は`docs/v{next}/TBD.md`へ書き残し、暫定判断で続行する。
ユーザーが回答を追記したうえで本スキルを再起動すれば回答済み事項を反映して続行できる。

## 1. 位置付けと前提

入力前提:

- `docs/v{next}/{theme}.md`（作業テーマドキュメント）が既に存在する
- `docs/v{next}/OVERVIEW.md`（次版総合ドキュメント）が既に存在する
- 上記の配置は`agent-toolkit:spec-driven`配下`references/spec-driven-framework.md`の既定または
  プロジェクト指定に従う

起動方法:

- ユーザーは作業テーマ名を引数または本文で指定する
- 指定がない場合は`docs/v{next}/`配下の作業テーマドキュメントを全て走査対象とする
- 起動時に入力前提を満たさない場合はTBD.md末尾に「## 停止理由」章を追記して終了する

## 2. ユーザー確認規範のオーバーライド

本スキル実行中は次の確認契機をTBD.md追記＋暫定判断＋続行へ置換する。

- `agent.md`「品質最優先」節および「ユーザーとともに考える」節の確認要求全般
- `claude-code.md`「auto mode下での挙動」節「迷ったら作業を止めて確認」
- `agent-toolkit:plan-mode`の要件対話・認識合わせ・恒久化検討・リファクタリング検討の各ユーザー確認
- `agent-toolkit:plan-impl`「2.5サブエージェント完了報告の検収」のうちユーザー確認を要する分岐

オーバーライド対象外（本スキル中も従来規範どおりに停止する）:

- 公開インターフェース変更でユーザー合意が無いもの
- 破壊的操作・外部送信（`git reset --hard`・`rm`・`git push`・外部APIへの送信など）
- 検証コマンドの恒常的失敗（リトライで解消しない正当な失敗）
- 本スキルの実行前提が崩れる状態（`docs/v{next}/`不在など）

上記に該当する場合は工程をその時点で停止し、TBD.md末尾に「## 停止理由」章を追加して終了する。

## 3. ワークフロー

1. 起動引数または走査結果から対象テーマ集合を確定する
2. `docs/v{next}/TBD.md`を読み込み、回答済み質問の反映方針を確定する。
   未作成ならテンプレで新規作成する
3. テーマごとに「4. テーマ実装ループ」を実行する
4. 全テーマ完了後、TBD.mdの未回答件数と冒頭リストを最終メッセージで提示する

## 4. テーマ実装ループ

1. 簡易計画ファイル`~/.claude/plans/spec-driven-impl-{theme}.md`を生成する。
   書式は`agent-toolkit:plan-mode`配下`references/plan-file-guidelines.md`のH2構成に従う
2. `agent-toolkit:plan-impl`を参照呼び出しし、タスク分解→実装委譲→検収→検証→コミットを実施する。
   `EnterPlanMode`/`ExitPlanMode`は呼び出さない
3. 各工程でユーザー確認が必要になった都度、TBD.mdへ質問ブロックを追記し暫定判断で続行する
4. TBD.mdに既存の回答済み質問があれば、暫定採用との一致／不一致を判定する。
   不一致なら当該箇所の改修タスクを追加する

## 5. TBD.md書式

章構成: 冒頭に本ファイルの説明文、`## {theme name}`配下に`### Q{n}: {要約}`ブロックを並べる。
最終的に未完了テーマが残った場合は末尾に`## 停止理由`章を追加する。

質問ブロック雛形:

````markdown
### Q{n}: {一行で要約}

- 発生工程: {計画|実装|検証|コミット}
- 関連箇所: `path/to/file` または `docs/v{next}/{theme}.md`の節名
- 背景: {何が決められず何を仮置きしたか}
- 選択肢:
  - A: {...}
  - B: {...}
- 暫定採用: {ラベル}（理由: {根拠}）
- 回答:
  <!-- ユーザーはこの行以降に回答を追記する -->
````

回答欄判定: 直後のHTMLコメント行を除き非空行があれば「回答済み」と判定する。
非空行がなければ「未回答」と判定する。

## 6. 停止判定

「2. ユーザー確認規範のオーバーライド」のオーバーライド対象外に該当する事象を検知した場合、
当該テーマの工程を即時停止し、TBD.md末尾に`## 停止理由`章を追加する。

`## 停止理由`章には次を記録する。

- 停止契機（公開IF変更・破壊的操作・検証失敗・前提崩れのいずれか）
- 原因の要約
- 未完了テーマ名の一覧
- 再開時にユーザーへ確認したい事項

停止後は他テーマへ進まず本スキルを終了する。
