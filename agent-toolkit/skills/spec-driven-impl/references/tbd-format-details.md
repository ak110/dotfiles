# TBD.md書式詳細

`agent-toolkit:spec-driven-impl`スキル本文「5. TBD.md書式」節から参照する重複防止規定とユーザー指摘の追記欄雛形。

## 重複防止

新規Qブロック追記前に、同テーマ配下（`## {theme name}`配下）の既存Qブロックを走査して重複有無を確認する。

- 要約・関連箇所・背景がほぼ同一の質問が存在する場合は新規Qブロックを追加せず、既存ブロックの末尾サブバレットへ`- 検知回数: {回数}`を更新する
- E2Eテスト・動作確認などの環境制約由来で実行不能な事象は、サブエージェント実行のたびに同一原因で発生し得るため原因単位で1Qブロックへ集約する
- 重複判定基準は関連箇所のパスまたは節名が一致し、かつ背景・暫定採用の要旨が同義であること
- 同一原因事象を3回以上検知した場合は重複追記を打ち切り、サブエージェントは`needs_escalation`を返してメイン側調停（`4.5 エスカレーション受領時の調停`節）に引き渡す

## ユーザー指摘の追記欄

質問ブロックと並置できる形でユーザーが任意のタイミングで指摘を書き込めるよう、TBD.mdの末尾に`## ユーザー指摘`節を設ける。
雛形は次のとおり。

````markdown
## ユーザー指摘

### U{n}: {一行で要約}

- 関連箇所: `path/to/file` または `docs/v{next}/plans/{theme}.md`の節名
- 指摘内容:
  <!-- ユーザーはこの行以降に指摘を記入する -->
- 反映方針: {サブエージェントが計画ファイル巡回時に記入する}
````

サブエージェントは計画ファイル巡回ループの冒頭で本節を走査し、未対応の指摘について反映方針を確定する。
対応完了後も履歴として削除しない。
