# サブエージェントエスカレーション受領時の調停

サブエージェントが`needs_escalation`で返した場合、本節の手順でメイン側調停を実施する。
対象はTBD.md追記のみで解消しない事象に限る。

1. 受領情報の確認: サブエージェントの完了報告から
   「エスカレーション原因」「該当ステップ」「TBD.mdの集約Qブロック識別子」を抽出する
2. 原因分類: 次の3類型へ分類する
   - 環境制約系（並列実行不可・外部サービス未起動・権限不足等）
   - 計画不整合系（前段テーマ未完了・前提仕様の食い違い等）
   - 入力不足系（テストデータ不在・設定値の未定義等）
3. 分類別の調停手順を実施する
   - 環境制約系: 当該対処を専門に扱う新規`spec-driven-implementer`を別途起動して対処コミットまで完遂し、
     原テーマの`spec-driven-implementer`を再起動して続行する。
     対象は単独実行で解消する事象と、原テーマ計画ファイル範囲外への対処
    （既存バグの修正・前提環境の整備等）が必要な事象を含む
   - 計画不整合系: 当該計画ファイルの`## 変更内容`または依存テーマ計画ファイルを修正してから再起動する
   - 入力不足系: 必要素材を準備または取得し、計画ファイルへ反映してから再起動する
4. 調停結果の記録: TBD.mdの当該集約Qブロックの`回答`欄に調停内容と再起動結果を記入し、Qブロックを「回答済み」化する
5. 再起動可否: 再起動でも同一原因が3回以上発生した場合は調停不能扱いとせず、
   原因分類の選択肢（環境制約系・計画不整合系・入力不足系）を全て再評価する。
   試行履歴はTBD.mdの当該集約Qブロックへ追記する。
   全分類を試行しても技術的に解消不可能と確定した場合のみ最終メッセージへ記録し次テーマへ進む
  （停止判定の停止契機には該当しない）

調停は技術的な解消が見込める範囲に限定し、ユーザー確認規範のオーバーライド節を覆さない。
ユーザー判断を要する事項（`references/qblock-templates.md`「ユーザー作業必須雛形」適用範囲規定）は
TBD.mdへ残して次テーマへ進む。

調停完了後、解決済みとなったQブロック・解決済み発見事項・確定済み暫定判断は次サイクルへ持ち越さず、
TBD.md・計画ファイル本文・テーマ仕様・OVERVIEW.mdから削除して最新化する。
残置対象は未解決事象および確定根拠として必要な記録に限定する。
