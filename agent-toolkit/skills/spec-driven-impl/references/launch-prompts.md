# spec-driven-implementer起動プロンプトテンプレート

`agent-toolkit:spec-driven-impl`スキル本文「サブエージェント起動プロンプト」節から参照する3種テンプレート。

## 通常モード

```text
以下の計画ファイルを`agent-toolkit:plan-impl`スキルに従って実装してください。

- 計画ファイル: `docs/v{next}/plans/{theme}.md`
- 多論点計画の場合は`plan-impl`のタスク分解を論点単位で細かく行ってください
- 実装過程でテーマ仕様（`docs/v{next}/{theme}.md`）・次版総合ドキュメント（`docs/v{next}/OVERVIEW.md`）・計画ファイル側の追記・修正が必要と判明した場合は、テーマ仕様→計画ファイル→OVERVIEW.md→コードの順で反映してください（コード変更を先行させない）
- `docs/v{next}/OVERVIEW.md`の「次に行うべき作業」節をテーマ実装結果に合わせて更新してください（完了テーマの除去・新発覚事項の追記）。当該更新は計画ファイルのコミットと同一コミットに含め、計画ファイル`## 変更内容`の対象ファイル一覧へOVERVIEW.mdを明示的に追記してください
- 完了報告には「OVERVIEW.md『次に行うべき作業』節を更新したか」項目を含めてください
```

## 段1（並列実装専従モード）

```text
以下の計画ファイルを`agent-toolkit:plan-impl`スキルに従って実装してください。ただし本起動は並列実装専従モード（段1）であり、検証・OVERVIEW.md編集・`git commit`の実行は禁止します。

- 計画ファイル: `docs/v{next}/plans/{theme}.md`
- 多論点計画の場合は`plan-impl`のタスク分解を論点単位で細かく行ってください
- 実装過程でテーマ仕様（`docs/v{next}/{theme}.md`）・計画ファイル側の追記・修正が必要と判明した場合は、テーマ仕様→計画ファイル→コードの順で反映してください（コード変更を先行させない）。OVERVIEW.mdの更新は段2の検証・コミット専従サブエージェントが行うため、本サブエージェント内では実施しないでください
- 完了報告には「OVERVIEW.md更新の要否と追記内容案（完了テーマ除去・新発覚事項追記）」項目を含め、段2サブエージェントがOVERVIEW.md更新時に参照できるようにしてください
- 完了報告には「実機検証・VRT更新などメイン側または段2で実施すべき引き継ぎ事項」項目を含めてください
  - 並列実行制約や作業ツリー全体への影響で本サブエージェント単独では実行不能な検証・確認は、本項目で呼び出し元へ返却してください
- 検証コマンドの実行および`git commit`は段2の検証・コミット専従サブエージェントが行うため、本サブエージェント内では実行しないでください
- 完了報告には編集したファイルパス一覧を含めてください
```

## 段2（検証・コミット専従モード）

```text
以下の計画ファイルを`agent-toolkit:plan-impl`スキルの検証・コミット工程に従い完遂してください。実装は段1で完了済みです。

- 計画ファイル: `docs/v{next}/plans/{theme}.md`
- 段1で編集されたファイルパス一覧: {paths}
- 段1で生成されたOVERVIEW.md追記内容案: {overview_diff_proposal}
- 段1から引き継いだ実機検証・VRT更新等の事項: {handover_items}
- 当該テーマの`## 実行方法`に従う検証コマンドを実行してください
- OVERVIEW.mdの「次に行うべき作業」節を上記追記案で更新してください（完了テーマの除去・新発覚事項の追記）。当該更新は本テーマのコミットへ含めてください
- `git commit`を作成してください（pushは行わない）
- 完了報告には「OVERVIEW.md更新の実施」「検証結果」「作成コミットのSHA」を含めてください
```
