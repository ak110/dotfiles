# 肯定形優先の記述 変換テンプレ集

本ファイルは`agent-standards/SKILL.md`「肯定形優先の記述」節から参照する変換テンプレ集とする。
規範文書の禁止孤立表現を推奨事項付きの肯定形式へ書き換える書式を統一する目的で利用する。

## 変換テンプレ

### 節タイトル

禁止形の節タイトルは、実施すべき動作を主体に置いた形へ書き換える。

- before: `### private関数の直接テスト禁止`
- after: `### private関数はテストしない（公開インターフェース経由でテストする）`

### 単独の禁止列挙

代替手段を推奨事項として先置きし、禁止対象を補足で述べる。

- before: `Xを避ける`
- after: `Xの代わりにYを採用する。Xは<理由>のため避ける`

### 機械チェックの回避禁止

原因側の修正を推奨事項として明示する。

- before: `除外設定を使わない`
- after: `原因側修正で対応する。除外設定は<例外条件>の場合に限り採用可`

### スコープ拡大禁止

推奨する対処を先置きし、実施義務を明示する。

- before: `スコープ超過を理由に不対応にしない`
- after: `発見時点で同一計画に対処項目として組み込む。スコープ超過は不対応の根拠にしない`

### 破壊操作禁止

事前確認を経る条件を明示する。

- before: `〜を実行しない`
- after: `〜する場合は事前確認を経てから実行する`

## 適用ガイドライン

- 節タイトルは動詞主体で書き換え、目次・トリガー時に該当時の行動指針が読み手へ伝わる形式とする
- 単一節内で禁止と推奨をペアで併記する
- 既存の禁止規定は改訂時に本テンプレへ照合して更新する
