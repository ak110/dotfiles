# 既存不良の品質スイープ

## 目的とスコープ

ユーザー手動起動を契機に、既存プロジェクト全体のファイルを対象として
`plan-impl-reviewer`観点（コード単体品質・ドキュメント単体品質・日本語表現）の既存違反を網羅的に検出し、
発見した既存不良を同セッション内で修正する。

`agent-toolkit:overhaul-project`は改善観点を広く取る（コード改善・ドキュメント整備・足元整備）。
本スキルは`plan-impl-reviewer`が担う3観点に限定し、レビュー駆動の修正サイクルを繰り返す。

## 起動条件と前提

- ユーザー手動起動専用（`/agent-toolkit:quality-sweep`）
- plan-mode必須。`agent-toolkit:plan-mode`を呼び出す（手順は「進め方」手順1を参照）
- 品質基準は呼び出し先スキルへ委ねる
- 修正で公開インターフェース変更などの破壊的変更が発生する場合、計画段階でユーザー確認を得る

## 対象範囲と分割方針

対象ファイルをカテゴリで分類してから`claude`サブエージェントへ分担する。
分割の判定基準は`agent-toolkit:careful-review`「`plan-impl-reviewer`の分割起動」節をSSOTとして参照する。
件数閾値・カテゴリ統合条件・自動bumpファイルの扱いは当該節の規定に従う。

カテゴリ分類は次の3種で、`careful-review`規定と同一とする。

- コード・テストコード
- 一般ドキュメント
- コーディングエージェント向け文書

## 進め方

1. `agent-toolkit:plan-mode`を呼び出し計画ファイル作成プロセスへ入る
   （plan mode外の場合は先に`EnterPlanMode`でplan modeへ移行する）
2. 計画ファイルで以下を確定する
   - 対象ディレクトリと除外パス
   - カテゴリ別の対象ファイル件数見積もりと分担単位
   - 検証コマンド（プロジェクトで定められたチェック実行コマンド）
   - コミット件名案
3. `ExitPlanMode`後、本スキル本体が「分担サブエージェント（claude）の起動」節に従い分担サブエージェントを並列起動する
4. 全分担サブエージェント完了後、`git status --short`・`git diff --stat`で作業ツリー実態を照合する
5. 「最終検証とコミットサブエージェント（claude）の起動」節に従い最終検証サブエージェントを起動する

## 分担サブエージェント（claude）の起動

`claude`エージェントタイプで起動する（`plan-implementer`は使わない）。
担当ファイル群が独立する2件以上は`run_in_background=true`で並列起動する。
起動前に各分担の編集対象ファイル集合を点検し、重複があれば分担境界を再設計する。

起動プロンプトの雛形:

```text
以下のタスクを実装してください。

- 計画ファイル: `{計画ファイルパス}`
- 担当ファイル群: `{ファイルパス列挙}`
- 担当カテゴリ: `{コード・テストコード | 一般ドキュメント | コーディングエージェント向け文書}`

## 作業内容

1. 担当ファイル群を対象として`plan-impl-reviewer`サブエージェントを起動する
   - 対象範囲は「既存ファイル全体（差分起因欄は`指摘なし`で返却し、既存違反欄に全件を列挙する）」と明示する
2. `plan-impl-reviewer`の指摘（既存違反欄）を統合する
3. 重大度`致命的`・`重大`の指摘を全件修正する。重大度`軽微`は呼び出し元の判断で取捨する
4. 修正範囲は担当ファイル群に限定する

## 制約

- コミット・pushは実施しない（最終検証サブエージェントが担う）
- 作業ツリー全体へ影響する操作は禁止
  （該当例: `git stash`・`git checkout`・`git reset`・`git clean`・引数なしの`pre-commit run`）
- 担当ファイル外への変更は実施しない（必要が生じた場合は完了報告に記録して呼び出し元へ返す）

## 完了報告要件

- 修正したファイルパスと変更内容の要約
- 取り下げた指摘の件数と取り下げ理由
- 担当ファイル外への変更が必要と判断した事項（あれば）
```

## 最終検証とコミットサブエージェント（claude）の起動

`claude`エージェントタイプで1件起動する。
起動プロンプトの雛形:

```text
以下の検証とコミットを実施してください。

- 計画ファイル: `{計画ファイルパス}`
- 検証コマンド: `{計画ファイル記載の検証コマンド}`
- 件名案: `{計画ファイル記載の件名案}`

## 作業内容

1. 検証コマンドを実行し、警告・違反を全件確認する
2. 警告・違反は根本対応で解消する。解消後に再検証する
3. 全件greenを確認したら、`agent-toolkit:commit`規定に従ってコミットを作成する
   - 件名は計画ファイル記載の件名案を採用する
   - 本文には修正したカテゴリ・対象ファイル数・主な指摘内容を記述する

## 制約

- pushは実施しない（ユーザー指示後に呼び出し元で実施する）
- 機械チェックを回避する目的のlint抑制・テストskipは禁止
```

## 既存スキルとの連携

- `agent-toolkit:plan-mode`: 「進め方」手順1で呼び出す
- `agent-toolkit:careful-review`: 分割閾値の定義元。分担方針を確定する時点で参照する
- `agent-toolkit:commit`: 最終検証サブエージェントのコミット作成時
- `plan-impl-reviewer`（サブエージェント）: 各分担サブエージェント内で呼び出す
