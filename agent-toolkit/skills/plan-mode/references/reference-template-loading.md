# 6-1. 参照テンプレート読み込みの詳細手順

`plan-file-guidelines.md`「6-1. 参照テンプレート読み込み」節の詳細規定。

計画ファイルの初版Write直前に本ファイルをReadで読み込む（未実施時はブロックされるため簡易対応でも省略しない）。
対象ファイル一覧が`agent-toolkit:agent-standards`「文書サイズ上限」節の対象ファイルを含む計画を対象とする。
新規節・大幅追記は目安200行以内に収める設計を優先する（220行到達後の行数調整は実装時の後戻りコストが大きいため）。
対象ファイル全件は拡張子（`.md`・`.md.tmpl`・`.py`・`.json`等）を問わず現行の`wc -l`実測値を
`## 調査結果`配下の`### 対象ファイルの現状`H3へ列挙する（改訂後見込み行数は`.md`・`.md.tmpl`のみ併記する）。
220行超過ファイルはscratchpad配下の一時ファイルへ追記文面案と縮減対象文面を実際に貼り込み`wc -l`で実測する。
追記量と縮減量の相殺を`### エージェント判断`へ必須記載する（見た目行数と実物理行数の乖離を計画段階で解消するため）。
算定規則は削除・統合・見出し挿入（前後空行含む）の実物理行数とし、単文内の文言短縮は寄与しない。
縮減対象の選定では規範述語（「〜しない」「〜とする」等）を含む文の削除・弱体化をせず、意味を保存した統合・冗長表現の圧縮で行数を確保する。
220行超過ファイル対象時は`## 変更内容`配下へ`#### 縮減対象（<ファイル名>）`H4を配置する。
対象は`extract_addition_reduction_blocks`の`replacement_pair_count`・`reduction`が共に0の場合に限る。
H4ブロックには削除文言案を`text`フェンス形式で明示し（`check_wc_projection.py`が不在・不足を警告）、
`[現行]`/`[置換後]`ペアと`[削除根拠]`ペアの行数差は同関数により集計され実測値と一致する。

220行超過ファイル対象で追記量と縮減量を相殺する`[現行]`/`[置換後]`ペアを組む場合、圧縮対象の既存記述が縮減根拠4類型のいずれかに該当することを圧縮案確定前に検証する。
4類型の定義SSOTは`agent-toolkit:trim-agent-docs`「縮減根拠4類型」節とする。
該当類型を`### エージェント判断`へ明記する。
該当類型が見つからない場合は行数調整目的の機械的圧縮を採用しない。
代替として対象ファイル自体の見込み行数超過を許容するか`references/`分離を検討する。
見込み行数超過を許容する場合、既存規定の`#### 縮減対象（<ファイル名>）`H4配置義務は本項の例外として免除する（H4は縮減案を伴う場合の書式のため）。
行数調整のみを目的とした文結合・句読点省略・1文1行規定違反となる文統合は圧縮案から除外する。
