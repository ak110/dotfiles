# 参照テンプレート読み込みの詳細手順

`plan-file-guidelines.md`「6-1. 参照テンプレート読み込み」節の詳細規定。
文書サイズ上限（トリガー行数・ハードリミット・判定タイミング・判定主体）のSSOTは
`agent-toolkit:agent-standards`「文書サイズ上限」節とする。本節では計画時の適用手順のみを扱う。
改訂後の見込みがトリガー行数を超過する対象ファイルでは、縮減案の確定前に縮減根拠4類型の検証手順を適用し、
計画段階での行数超過見落としを防ぐ。ハードリミット到達後の行数調整は実装時の後戻りコストが大きいため、
計画段階でトリガー行数以内に収める設計を優先する。
計画時・計画レビュー時は行数の細かい計算・実測にコストをかけない。

対象ファイル一覧が`agent-toolkit:agent-standards`「文書サイズ上限」節の対象ファイルを含む計画を対象とする。
対象ファイル全件は拡張子（`.md`・`.md.tmpl`・`.py`・`.json`等）を問わず現行の`wc -l`実測値を
`## 調査結果`配下の`### 対象ファイルの現状`H3へ列挙する。

`.md`・`.md.tmpl`対象ファイルは、現行行数と追記量の概算を比較する。
`agent-toolkit:agent-standards`「文書サイズ上限」節のトリガー行数超過が見込まれる場合、
`## 変更内容`配下へ`#### 縮減対象（<ファイル名>）`H4を配置し縮減対象節と移設先を明示する。
概算は目視での大まかな見積りで足り、scratchpadへの仮適用・`wc -l`実測・追記量と縮減量の相殺照合は行わない。
H4ブロックには削除文言案を`text`フェンス形式で明示する。
算定規則は削除・統合・見出し挿入（前後空行含む）の実物理行数とし、単文内の文言短縮は寄与しない。
縮減対象の選定では規範述語（「〜しない」「〜とする」等）を含む文の削除・弱体化をせず、
意味を保存した統合・冗長表現の圧縮で行数を確保する。

縮減対象を確定する前に、圧縮対象の既存記述が縮減根拠4類型のいずれかに該当することを検証する。
4類型の定義SSOTは`agent-toolkit:trim-agent-docs`「縮減根拠4類型」節とする。
該当類型を`### エージェント判断`へ明記する。該当類型が見つからない場合は行数調整目的の機械的圧縮を
採用せず、`references/`分離を検討する。

計画段階では`agent-toolkit:agent-standards`「文書サイズ上限」節のハードリミットの厳密な事前充足確認は求めない。
