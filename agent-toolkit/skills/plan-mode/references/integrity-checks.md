# 計画ファイル整合性チェック

`agent-toolkit:plan-mode`スキル工程6-2の整合性チェックで適用する観点を節ごとに定義する。
関連スキル・サブエージェントが参照する節名は本ファイルの見出しを典拠とする。

## ユーザー発話・提示素材との照合

`## 背景`の許容情報源はユーザー発話・対話確認内容と提示素材原文に限定する。
提示素材は原文転記する。
調査結果・既定値・推測・エージェント解釈は`### エージェント判断`へ移動する。

## 計画文内・他ファイルとの整合

サブエージェント担当部分:

- 計画ファイル本文全域で同一概念を指す用語が複数表現で混在しないか照合する
- `## 変更内容`の対象ファイル一覧チェックボックス項目と各ファイル変更方針H3見出しが完全に対応するかを双方向照合する
- `## 調査結果`で影響を受けるとした既存ファイルが対象ファイル一覧に網羅反映されているかも併せて照合する
- 追記文面の表記が追記先の既存表記と整合しているかも併せて照合する
- 計画ファイル本文の追記文面と既存規範文書の間で同一概念の判定基準を2箇所以上に記述する場合、
  `grep -rn`で全箇所を抽出して文言完全一致を照合する
  - 代替案として定義元1箇所へ集約し、読取側は委譲記述（「判定基準は定義元に従う」相当）のみへ統一する形を優先する
  - やむを得ず読取側で再掲する場合は定義元と一字一句同一の表現に揃える
- `## 変更履歴`に記録された方針転換・修正内容が`## 変更内容`配下の文面・対象ファイル一覧・差分指示・サブエージェント起動プロンプト文面に同期反映されているか照合する

メイン側担当部分:

- 追記・改訂対象トピックに関連する別ファイルの記述（集約先案内文・対象範囲記述・列挙構造・節ラベルの転記箇所・典拠先記述・docstring内の同義説明など）に追従更新の要否を`grep -rn`で網羅確認する
- ファイル一覧で得たファイルは本文まで読み込んで対応関係を確認する

## 機械チェック適合性

計画ファイル本文全域が実装後の機械チェック（sentence-length・line-length・口語表現・漢字7字連続）を通過する形か確認する。
`## 背景`配下の原文転記領域は本節の例外として違反を許容する。
`## 変更内容`・`## 対応方針`配下のコードブロック内文面はagent.md「日本語の品質を保つ」節の上位原則と`agent-toolkit:writing-standards`「Markdown記述スタイル」節で点検する。
diff `+`側文面は計画ファイル本体の機械チェックでは実質的に検査対象外となるため、`agent-toolkit:plan-mode`スキル本文の工程7ステップ1にある一時ファイル機械チェック手順で検査する。

## 既存規範の周辺品質の再点検

既存規範の改訂・拡張時は周辺の既存文面の日本語表現を再点検し、不自然な表現を踏襲したまま追記しない。
複数概念を1述句へマージする場合は各概念の手順・条件・対象の差異が脱落していないか照合し、差異があれば分割表現を保持する。
新ルール・規範（計画策定プロセス自体の試算手順・調査手順・整合性チェック手順を含む）を追加または改訂する場合、当該規範を計画ファイル本文にも適用して整合させる。
新規規範を追加する場合、同節・隣接節の既存箇条のうち新規規範の対象事象に該当する記述（特に分岐記述・条件分岐・停止判定）を全件抽出し、新規規範で置換・統合する形になるか確認する。
新規箇条として既存箇条と並置すると、対象事象が新旧両方の判定経路に該当し、重複処理や取りこぼしが生じる。

## 編集対象スキル固有規定の事前適用

編集対象スキル・上位スキルの本文から計画段階の必須記載事項・事前適用規定を全件抽出し、計画ファイル本文の該当セクションへ反映する。
`agent-toolkit/rules/agent.md`「スキルの事前呼び出し」節のトリガーマッピングから編集対象ファイル種別の必須スキルを全件列挙し`## 実行方法`の呼出スキル一覧へ反映する。
対象リポジトリの`AGENTS.md`・`CLAUDE.md`に書かれた呼出元スキル完了後の後続コマンドを`## 実行方法`末尾へ反映する。

`agent-toolkit:agent-standards`「文書サイズ上限」節の対象ファイル（`agent.md`・`claude-code.md`・各`SKILL.md`・サブエージェント定義・`references/`配下など）の検査。
当該節の対象ファイルが計画ファイル`## 変更内容`の対象ファイル一覧へ含まれる場合は、計画ファイル本文の以下2点を機械的に検査する。

- `### エージェント判断`に各対象ファイルの改訂後の試算行数記載が存在するかを検査する
- `## 変更内容`に物理行数の目標値および`wc -l`確認手順記載が存在するかを検査する
- いずれかが欠落している場合は重大指摘として報告する

## 担当分担

各節の担当先を次の通り定める。

- メイン側担当:「ユーザー発話・提示素材との照合」全体と、「計画文内・他ファイルとの整合」のうち横断`grep -rn`確認・関連ファイル本文の通読部分
- `plan-integrity-checker`サブエージェント担当: 上記以外の節（「計画文内・他ファイルとの整合」の計画ファイル内部照合部分・「機械チェック適合性」・「既存規範の周辺品質の再点検」・「編集対象スキル固有規定の事前適用」）

サブエージェント担当節とメイン側担当節の対応関係は呼び出し元コンテキスト参照の要否で分かれる。
