# 計画ファイル整合性チェック観点

`agent-toolkit:plan-mode`スキル工程6-2の整合性チェックで適用する観点A〜Jの定義。
観点記号・名称は本ファイルを典拠とし、関連スキル・サブエージェントが参照する呼称もこれに統一する。
矛盾は関連セクション間で一致するまで解消する（観点は例示集で網羅規定ではない）。

- 観点A（ユーザー発話・提示素材との照合）:
  `## 背景`の許容情報源はユーザー発話・対話確認内容と提示素材原文に限定し、提示素材は原文転記する（調査結果・既定値・推測・エージェント解釈は`### エージェント判断`へ移動する）
- 観点B（変更内容とエージェント判断の相互照合）:
  `## 変更内容`の対象ファイルと`### エージェント判断`の方針の双方向整合、`### 恒久化・リファクタリング内容`の結論記載（対象無は「なし」明記）、永続化先・周辺対応の`## 変更内容`反映を照合する。
  `## 調査結果`で影響を受ける既存ファイルとして列挙したものが`## 変更内容`の対象ファイル見出しへ網羅反映されているかも双方向照合する（「〜参照する記述があり、〜後も維持する必要がある」「〜未記載である」のような既存影響箇所への言及を含む）。
  数値・識別子・規定値・状態語（件数・名称・節名・フラグ表記・引数既定値などの利用者観測値）の変更・分化時は計画ファイル全文と参照先ファイルを`grep -nE`で再走査し、算出式・結果値・他セクションを追従更新する。
  `## 変更内容`のdiff +側で明示された新規規範要素を行数縮減のため1項目へマージする場合は、マージ前後で要素単位の対応関係を照合し最終要素の脱落を検出する
- 観点C（計画対象ファイル内の方針整合）:
  `## 変更内容`で計画対象ファイルへ新方針・新節を追記する場合、同一ファイル内の既存方針と文書冒頭の目次的記述（章節一覧・サマリーテーブル・変更履歴の対応表）との整合を照合する。
  同一文書内のセクション間対応関係（能力列挙の「含むもの」「スコープ」等と受け入れ基準など観測可能な条件を記述するセクションとの対応）も照合対象に含める。
  追記位置の前後数段落に同種論点（同一対処を異なるトリガー条件で規定するもの）が既に記述されていないかを確認し、追加文と既存文の役割の分担および統合可否を照合する。
  衝突時は既存方針の改訂と目次的記述の追従更新を`## 変更内容`へ含める。
  計画文面のコードブロックに実装案コードを提示する場合、同一モジュール内の既存類似関数の実装方式と整合するかを照合する
- 観点D（計画文面の機械チェック適合性）:
  計画ファイル本文全域が実装後の機械チェック（sentence-length・line-length・口語表現・漢字7字連続）を通過する形か（`## 背景`配下の原文転記領域は観点A既定の例外）。
  括弧書きを含む長文や半角識別子を多用する行は計画策定時点で分割し、口語禁止語は`agent-toolkit:writing-standards`「口語表現チェック」節名で間接参照する。
  diffの+側文字列の127幅line-length・漢字7字連続・口語表現は機械チェックで実質的に検査対象外となるため計画策定者が手動で点検する。
  計画文面案の確定前に`agent-toolkit:writing-standards`配下`references/textlint-violations.md`の頻出違反パターンを参照し先回り抑止する
- 観点E（計画文面内部・追記先ファイルとの用語統一）:
  計画ファイル本文内で同一概念を指す用語が複数表現で混在していないか照合し、混在時は統一する。
  追記・改訂を含む変更では追記文面の表記が追記先の既存表記と整合しているか`grep -rn`で確認する。
  単一計画内で複数の新規ファイルを生成する場合は新規ファイル間のプレースホルダー表記の整合も確認する
- 観点F（既存規範の改訂箇所周辺における品質・規範違反・意味脱落の再点検）:
  既存規範の改訂・拡張時は周辺の既存文面の日本語表現を再点検し、不自然な表現を踏襲したまま追記しない。
  複数概念を1述句へマージする場合は各概念の手順・条件・対象の差異が脱落していないか照合し、差異があれば分割表現を保持する
- 観点G（ファイル間対応関係の追従整合）:
  追記・改訂対象トピックに関連する別ファイルの記述で追従更新の要否を`grep -rn`で網羅確認する。
  対象カテゴリは集約先案内文・対象範囲記述・列挙構造・観点や規範や節のラベル文言の転記箇所・典拠先記述とする。
  典拠先記述には移管元ファイルを典拠として名指す記述を含める（「〜で固定された呼称をそのまま用いる」「詳細は〜の観点Aに従う」など）。
  旧典拠パス・旧スキル名の名指し記述も`grep -rn`で網羅確認する。
  既存記述を別ファイルに委ねて省略・縮約する場合は省略内容が省略先で実際にカバーされているか確認し、未カバーなら省略を取りやめるか省略先へ先に追記する
- 観点H（計画文面の機械チェック非対象の文面品質）:
  `## 変更内容`・`## 対応方針`配下のコードブロック文面をagent.md「日本語の品質を保つ」節の上位原則と`agent-toolkit:writing-standards`「Markdown記述スタイル」節で点検する。
  コードブロック内の文面が1文1改行（句点ごとの改行）・1項目1文の規定に従っているかを点検する（sentence-lengthでは検出しないため計画段階で分割しておく）
  - 1物理行が127幅を超過する場合はサブバレット階層化または項目分割で予防し、継続インデントによる行折り返しは採用しない
- 観点I（新規・改訂ルールの計画ファイル本文への自己適用）:
  計画ファイル内で新ルール・規範（計画策定プロセス自体の試算手順・調査手順・整合性チェック手順を含む）を追加または改訂する場合、当該規範を計画ファイル本文にも適用して整合させる
- 観点J（対象編集スキル固有の事前適用規定の網羅的な事前適用）:
  編集対象スキル・上位スキルの本文から計画段階の必須記載事項・事前適用規定を全件列挙する。
  計画ファイル本文の該当セクションと1件ずつ照合し、未記載項目をすべて反映してから工程7へ進む
  - 抽出対象に`agent-toolkit:agent-standards`「メタ記述の禁止」節の4種違反パターンと「文書サイズ上限」節の物理行数の試算規定を含める
  - 工程2.5で起動した配置先スキルが指定する必須記載事項が計画ファイルへ反映済みかを照合する。
    対象例はbump種別の記載・bump実行ステップ・呼出スキルの列挙・固有の検証手順など
  - 抽出対象に`agent-toolkit/rules/agent.md`「スキルの事前呼び出し」節のトリガーマッピングを含め、編集対象ファイル種別から必須スキルを全件列挙し`## 実行方法`の呼出スキル一覧と照合する

観点A・観点Gはメイン側で実施するため、`plan-integrity-checker`サブエージェントの担当範囲には含めない。
