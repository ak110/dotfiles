# 新規規範または既存規範改訂を含む計画の点検手順

新規規範または既存規範改訂を含む計画では、計画ファイル作成前に以下を点検する。

## 規範対象範囲の網羅確認

新規規範の追加または既存規範の改訂時は、規範対象範囲のスクリプト群・
コード群・既存規範文書を網羅確認する。

- 対象語の同義・類義・近接表現を全件展開した`grep`または`rg`コマンドと、
  対象ファイル一覧・修正痕の有無を`## 調査結果`へ記載する
- 近接表現には件数表記（「N件」「N観点」「N項目」「N点」等）も含める
- スキャン対象は当該規範の適用範囲に含まれる全ファイルとする
- 横断スキャン時は`Explore`サブエージェントへ全文読了を委譲する
  - `grep`または`rg`のパターン検索単独では完結させない
- 修正痕の例
  - 不自然な文体・冗長な書き換え・元の論理関係を失った文面
  - 規範違反を伴う設計・誤った前提に依存する実装
  - 新設チェックリスト・要件列挙が既存対象実体に未適用のまま残存する状態
- 検出時の見直しは、規範改訂と同一コミットへ含めるか、
  改訂コミットに連なる連番コミットとして同セッション内で完遂する
- 出力フォーマット・必須記載項目・文書サイズ上限などの、
  構造的・形式的な規範の改訂時も対象に含む
- 遡及スキャン対象は、新規規範本文が判断軸として直接参照する成果物カテゴリに限定する
  - 例として、`apply-feedback`の判定対象なら過去の判定対症痕、
    コーディング規範ならコード抑制痕
  - 新規規範の例示部に登場する対象種別（lint抑制・型チェック抑制等）は、
    当該規範が直接管轄する成果物カテゴリ外にある古い抑制サイトはスコープに含めない

## 追記先節の選定

- 既存規範文書へ追記する場合は追加対象節の前後を読み込み、
  節タイトルが示す主題範囲と節本文の発火条件・対象が新規規範の発火条件・対象と整合するかを照合する
- 整合しない場合は他候補節との適合度を比較し最適節を選ぶ

## 既存条件節との整合確認

対象スキル内の既存条件節（並列起動条件・編集対象集合の無衝突条件・許容例外列挙など）と新規規範の組み合わせで
論理矛盾が生じないかを突合確認する。
観点は並列実行条件への影響・共有ファイル編集の競合可能性・既存の例外列挙の網羅性を損なう可能性とする。

## 機械検査スクリプトとの整合

計画ファイル形式・hook動作・自動チェック対象の規範を追加・変更する場合は、
機械検査スクリプト（`posttooluse.py`・`pretooluse.py`等）の検査対象を`grep`・`Read`で確認する。
機械検査吸収か対応コード変更を`### エージェント判断`へ明記する。

## 複数規範改訂の論理交差確認

同一計画内でN件（N≧2）の規範改訂を含む場合、C(N,2)=N(N-1)/2ペアを全件机上シミュレーションする。

- シミュレーション内容は「規範Aの条件成立時に規範Bの判定が矛盾なく成立するか」とする
- 確認結果を`### エージェント判断`へ記載する
- 交差確認の観点は条件式の整合・閾値の連動・発火タイミングの一貫性とする

## インライン形式での絶対の行番号参照の陳腐化点検

編集対象ファイル本文に対し`grep -nE '本(項目|節|文)L[0-9]+|L[0-9]+(-[0-9]+)?(記載|参照|配下)'`
等で同一ファイル内の絶対の行番号参照（`L<N>`形式）を検出する。
改訂による行番号のずれで参照値が陳腐化する場合は、意味参照（節名・項目冒頭文・対象識別子等）への
変換項目を`## 変更内容`へ組み込む。
検出対象は規範対象ファイルだけでなく、計画ファイル本文自身の絶対の行番号参照（`## 調査結果`の確定値を除く）も含む。

## 新規規範の文面に対するtextlint事前検証

新規の節見出し・規範本文・バレット項目の文面案は`## 変更内容`へ転記する前に、
計画作成段階で一時ファイル（scratchpad配下）へ出力する。
出力したファイルに以下のコマンドを適用する。

```sh
uvx pyfltr run-for-agent --commands=textlint,markdownlint,typos,colloquial-check --enable=colloquial-check <一時ファイル>
```

漢字連続違反・sentence-length違反・markdownlint違反・typos違反・口語表現違反の有無を確認する。
違反検出時は計画段階で文面を調整して再検証する。
違反回避と日本語表現の自然さが両立しない場合は当該事項を計画ファイル`### エージェント判断`へ記録してから
`## 変更内容`を確定する。
