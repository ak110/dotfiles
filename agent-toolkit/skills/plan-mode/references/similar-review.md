# 類似見直しの汎用定義

適用契機・深さ基準・記録先のSSOT。成果物側（コード・スキル・設定・CLIサブコマンド群等）の水平展開点検を指す。
`agent-toolkit/skills/agent-standards/references/feedback-review-common.md`「規範改善案の併記」
（規範文書側の抽象化）とは独立した観点である。両者は排他ではなく併用する。

## 適用契機

類似見直しは次のいずれかの契機で実施する。

- バグ対応: `bugfix-process.md`「2. 根本原因への対処」の実施時に適用する（既定の適用契機）
- フィードバック処理: 採用した全フィードバックへ`agent-toolkit:process-feedbacks`
  ステップ5「計画作成と実行」から適用する。対象はバグ起因のフィードバックに限定しない
- 通常計画: `agent-toolkit:plan-mode`工程5「リファクタリング検討」の標準観点として全計画へ適用する

## 深さ基準

- 母集団の列挙: 修正対象が属する同種構成要素のカテゴリ（CLIの全サブコマンド・全スキル・全hook・全テンプレート・全エージェント定義等）を特定し、メンバーを全数列挙する。
  同一ファイル・同一関数の近傍に閉じた点検は本基準を満たさない
- 悉皆点検: 列挙した全メンバーへ今回の指摘・修正と同一の観点を適用して点検する。
  複数ファイル横断時は`Explore`サブエージェントへの並列委譲を既定とする
- 記録: 点検した母集団・観点・結果（該当あり／なし・該当箇所一覧）を計画ファイルへ記録し、該当ありの箇所は同一計画の`## 変更内容`へ取り込む

## 記録先

- バグ対応計画: `### バグ調査結果`表の「類似見直し結果」欄
- フィードバック採用計画・通常計画: `## 調査結果`配下へ点検結果を記述し、
  該当ありの場合は`### 恒久化・リファクタリング内容`へも要約を併記する
