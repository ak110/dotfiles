# 計画作成プロセス改訂時の所要時間・厳格化点検

`norm-revision-checklist.md`「計画作成プロセス改訂時の所要時間・厳格化点検」節から参照されるSSOT。

`plan-mode`関連の工程・チェック・レビュー経路を追加する改訂では、
削減・軽量化の同時検討結果を計画ファイルへ明記し、追加のみの改訂を認めない。

- 追加対象が工程（`## 1.`から`## 7.`相当）・チェック項目（`check_plan_file.py`統合対象等）・
  レビュー経路（サブエージェント起動・codex継続呼び出し等）・再レビュー契機のいずれかに該当するかを判定する
- 該当する場合、同一改訂内で次のいずれかの削減候補を検討し、`### エージェント判断`へ検討結果を記載する。
  判定材料は「実装時修正コスト（後戻り・再設計を要するか、局所修正で解消できるか）」とし、
  局所修正で解消できる観点は削減候補として優先的に検討する
  - 既存工程・既存チェック項目・既存レビュー経路のうち、体裁・表記など致命度の低い観点の警告化
  - 既存の再レビュー契機のうち、機械チェックの再実行とplan-file-creatorによる自己照合で代替可能な契機の簡素化
  - 既存サブエージェント起動のうち、担当観点が他サブエージェントへ統合可能なものの整理
- 削減候補が存在しないと判断した場合はその根拠を`### エージェント判断`へ明記する
  （追加のみで削減候補が無いという結論自体は許容するが、無検討での追加は認めない）
