# 変更内容セクションの差分ラベルとフェンス配置

`plan-file-guidelines.md`「変更内容」節の詳細規定。

## 差分ラベル6種

コーディングエージェント向け文書の改訂タスクでは、置換・追記・削除のいずれかを明示し、
置換は対比形式、追記は最終文面、削除は現行文言と削除根拠をそれぞれ`text`コードブロックへ記載する。
差分ラベルは次の6種類とする。
`[現行]`は対比形式（置換前提示）と削除形式（削除対象提示）の双方で共用する。

- 既存節間挿入（置換前）: `[現行]`
- 既存節間挿入（置換後）: `[置換後]`
- 既存節への追加（隣接文言との対比を伴わない場合）: `[追記]`
- 新設ファイル全文: `[新設]`単独
- 既存ファイル全文置換: `[置換後（全文）]`
- 削除: `[現行]`と`[削除根拠]`の組

`[追記]`ラベルの使用条件を次のとおり定める。
追記量が既存節の隣接文言と1対1対応する形で表現できる場合は`[現行]/[置換後]`対比形式を必須とする。
隣接文言に依存しない既存節への追加のみを対象とする場合に限り`[追記]`ラベルを使用する。

## フェンス配置

各ラベルは`text`フェンス直後1行目にプレーンテキストで配置する（fence内側形式）。
textlint jtf-style/4.3.2大かっこのauto-fixが半角`[]`を全角`［］`へ変換し検査と衝突するため、fence外側配置は禁止する。
対比形式は隣接見出し行を`[現行]`、隣接見出し＋新規小節を`[置換後]`とし1回一致置換を可能にする。
構造化ファイル（JSON・YAML等）では親要素と開閉ブラケットを含める。
対象ファイルが`.py`等のPythonコードの場合、差分ラベル方式（`[現行]/[置換後]/[削除根拠]`）で書き、
差分ラベル外の平文フェンス（言語指定`python`または言語指定なしのフェンス）は避ける。
textlintがフェンス内のPython行を日本語文として誤検出し、`sentence-length`等の違反を報告するため。

## 機械適用対象

- `[現行]`/`[置換後]`対比と`[削除根拠]`は`check_wc_projection.py`が検査する
- `[追記]`は`check_wc_projection.py`と`check_plan_diff_gates.py`の直接検出対象とする
  （ラベル行は集計から除外し、フェンス本文をcolloquial-check・textlint検査へ渡す）
- `[新設]`・`[置換後]`・`[置換後（全文）]`は`check_plan_diff_gates.py`が検査する
- `[現行]`/`[置換後]`のfence外側配置は`check_plan_diff_gates.py`が検査する
  （対比ペア2種に限定する。textlint autofix全角化の観測実績が同ペアに集中するため）
- 全角化`[現行］`/`[置換後］`のfence外側出現も同スクリプトが検査する（同上の限定根拠）
- 各ラベル付きブロックは`### <ファイルパス>`H3配下へ配置する

## 記載範囲

記載する最終文面には既存の並列列挙要素・既存参照リンク・既存の代表例列挙を漏れなく含める。
本節をSSOTとし、`## 変更履歴`側の対象ファイルパス・節名アンカーは本節と一致させる
（`pretooluse.py`の`_check_plan_file_history_content_sync`が機械検査する）。

## 単体テストファイル追加

公開関数の追加・変更を含む計画では対応する単体テストファイルへの追加・修正項目を含め、
同値分割・境界値分析および正規表現の選択肢分岐（`|`）の各分岐でケースを抽出する。
