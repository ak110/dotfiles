# 工程5補足: 既存運用データの移行検討・別課題化の許容基準

`agent-toolkit:plan-mode`工程5「リファクタリング検討」から参照されるSSOT。

## 既存運用データ・運用仕組みの移行検討

運用中のシステム・仕組み（既存運用ディレクトリ・既存ファイル形式・既存設定ファイル・個人運用メモなど）の改修時は、
既存データを新仕様へ移行する手段を検討する。

- 既存データの新形式への変換手順（リネーム・ディレクトリ移動・スキーマ変換など）
- 旧形式の残置可否と互換性保持の必要性（git履歴に残せば作業ツリーから除去してよいか、利用側コードが旧形式を解釈できなくなる影響など）
- 移行検証手順の組み込み（移行前後の件数照合・サンプル抽出比較・移行後の動作確認）

移行が必要と判断した場合は移行手順を`## 変更内容`へ列挙し、`## 実行方法`の検証ステップ直前へ記述する。
不要と判断した場合は理由を`### エージェント判断`へ明記する。

## 別課題化の許容基準

発見した既存規範違反（意図的重複の同期注記欠落・SSOTルール違反・重複記述違反・命名整合違反など）は、
別課題化してよい場合を限定する。
別課題化を許容するのはSSOT境界の再設計・複数概念モデルの統合・公開インターフェース波及・
利用者向け構造改変を伴う重量作業に該当する場合のみとする。
frontmatter同期注記の追加・冒頭コメント文の追記・単一節内の重複解消など、
対象ファイル数と追加行数が有限で明示可能な軽量是正は本計画スコープへ取り込む。
