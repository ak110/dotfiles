# 既存規範文の短縮・言い換え時の並列列挙要素・根拠段落の照合

`norm-revision-checklist.md`「既存規範文の短縮・言い換え時の並列列挙要素・根拠段落の照合」節から参照されるSSOT。

既存規範文を短縮・言い換える形の改訂案を`## 変更内容`のdiff案に記述する場合、
既存原文の並列列挙要素（「A・B」形式、「A、B、C」形式など）を抽出し、
改訂後文面が全要素の意味範囲を保持できているか差分単位で照合する。

- 改訂前原文から並列列挙記号（「・」「、」「および」「または」等）で区切られた要素を全件列挙する
- 改訂後文面の対応語で全要素の意味範囲を保持できるか要素単位で判定する
- 意図的除外がある場合は`### エージェント判断`へ除外要素と除外理由を明記する
- 要素の意味的な等価性の判定はエージェント自己判定に委ね、判定困難時は原文の並列列挙をそのまま保持する。
  判定困難と判定した根拠は`### エージェント判断`へ記録する
- 既存規範文に数値・閾値・境界値（行数上限・件数制限・時間制限等）とその根拠段落（数値の由来・
  境界を分ける理由・トレードオフ説明）が併記されている場合、短縮・言い換え時の照合対象へ
  根拠段落を含める。根拠段落を欠落させる改訂案は採用せず、根拠を維持したまま短縮する代替案へ
  書き換える。維持困難な場合は`### エージェント判断`へ理由を明記する
