# テーマごとの実装計画作成

`agent-toolkit:spec-driven`で作成した次版ドキュメント（テーマ仕様）を入力として、テーマごとの実装計画ファイルを作成する。
ユーザー対話を伴い未確定事項を計画段階で確定させる。
本スキルはテーマ1件あたり`claude`サブエージェント2件（フェーズA・B）にplan-mode工程の実施を委譲する。
フェーズA完了時にメインがユーザーへ確認質問を発行し、回答をフェーズB起動プロンプトへ埋め込む。
plan-mode工程7の完了後（`ExitPlanMode`直前）で終了する。

## 1. 位置付けと前提

入力前提:

- `docs/v{next}/{theme}.md`（作業テーマドキュメント）が1件以上存在する
- `docs/v{next}/OVERVIEW.md`（次版総合ドキュメント）が存在する
- 上記の配置は`agent-toolkit:spec-driven`配下`references/spec-driven-framework.md`の既定または
  プロジェクト指定に従う

起動方法:

- 引数なしで起動する（`/agent-toolkit:spec-driven-plan`または`agent-toolkit:spec-driven`からの誘導）
- 起動時に入力前提を満たさない場合はユーザーへ報告して終了する
- 開始時の処理範囲確認やテーマ間の継続可否確認をユーザーへ問わず、未処理テーマがあれば全件を自動継続する

## 2. 対象テーマの確定

次版配置内の作業テーマドキュメントを走査し、対応する計画ファイル（`docs/v{next}/plans/{theme}.md`）が
未作成のものを処理対象とする。
処理対象テーマがなくなるまで、テーマ1件ごとに「3. ワークフロー」を繰り返す。
全テーマの計画ファイルが既に作成済みの場合は、その旨をユーザーへ報告して終了する。

## 3. ワークフロー

対象テーマ1件ごとに次の3段を実施する。
メインは段間で`AskUserQuestion`の発行とサブエージェント起動・完了検収のみを担う。

1. フェーズA起動: `claude`サブエージェント1件を起動し、plan-mode工程2前半〜2.7と暫定計画ファイル初版作成を委譲する
2. ユーザー確認: フェーズA完了報告に含まれる確認事項リストを整形し`AskUserQuestion`で発行する
3. フェーズB起動: ユーザー回答を埋め込んだ起動プロンプトで`claude`サブエージェント1件を起動し、
   plan-mode工程3〜7と暫定計画ファイル本体更新を委譲する

plan-mode本体の工程実施に対する読み替えは次のとおり。

- 計画ファイル配置は`docs/v{next}/plans/{theme}.md`へ読み替える。
  plan-mode規定の`~/.claude/plans/*.md`は使用しない
- `EnterPlanMode`は呼ばない（plan modeへは入らない）
- 工程8（`ExitPlanMode`と`plan-impl-executor`への引き継ぎ）は実施しない
- `docs/v{next}/plans/`ディレクトリが存在しない場合は計画ファイル作成前に作成する

工程7のcodexレビュー等の長時間処理が実行中の間は、レビュー結果を待たず次テーマのフェーズAを先行起動する。
レビュー結果は完了次第、対応するテーマの計画ファイルへ事後に反映する。
次テーマのフェーズAは現テーマのフェーズBと並走できる。

フェーズBは`claude`サブエージェント1件で完遂する。
フェーズBサブエージェントは`01-agent.md`「縮退表明は発行しない」項の規定に従う。

### フェーズA起動プロンプト

```text
次のテーマの計画ファイル初版を作成してください。

- 対象テーマ: `docs/v{next}/{theme}.md`
- 出力先: `docs/v{next}/plans/{theme}.md`

## 実施工程

`agent-toolkit:plan-mode`スキルをSkillツールで呼び出し、その規定に従いplan-mode工程2前半〜2.7と工程6の暫定初版作成までを完遂してください。
plan-modeの計画ファイル配置は`docs/v{next}/plans/{theme}.md`へ読み替えてください。
`EnterPlanMode`は呼ばないでください。
plan-mode工程2.5以前の調査でテーマ仕様（`docs/v{next}/{theme}.md`）側の追記・修正が必要と判明した場合は、
同セッション内でテーマ仕様ファイルも修正してください。

## 完了報告に含めるもの

- 暫定計画ファイルの保存パス
- ユーザーへの確認事項リスト（決定の依存関係順に並べる）
  各確認事項は対象論点・推奨回答・選択肢候補を含める
- 確認待ちの項目がない場合はその旨を明示する
```

### フェーズB起動プロンプト

```text
ユーザー回答を反映して計画ファイルを完成させてください。

- 計画ファイル: `docs/v{next}/plans/{theme}.md`
- 対象テーマ: `docs/v{next}/{theme}.md`

## ユーザー回答

{メインがAskUserQuestion回答を整形した本文}

## 実施工程

`agent-toolkit:plan-mode`スキルをSkillツールで呼び出し、その規定に従いplan-mode工程3〜7を完遂してください。
計画ファイル配置は`docs/v{next}/plans/{theme}.md`へ読み替えてください。
`EnterPlanMode`・`ExitPlanMode`は呼ばないでください。
工程7の整合性チェックは`plan-reviewer`サブエージェントとcodexレビューの並列起動を含めて完遂してください。
工程3〜7の過程でテーマ仕様（`docs/v{next}/{theme}.md`）側の追記・修正が必要と判明した場合は、
同セッション内でテーマ仕様ファイルも修正してください。

## 完了報告に含めるもの

- 計画ファイルの最終形に至った旨の確認
- 工程7で生じたレビュー指摘の反映結果
```

## 4. サブエージェントの役割分担

メインエージェントは段間の調整に専念する。
plan-mode工程の実施はフェーズA・Bの`claude`サブエージェントが担う。

- メインの責務: フェーズA起動・完了検収・`AskUserQuestion`発行・回答整形・フェーズB起動・完了検収・全テーマ完了後のコミット。
  メインは直接実装・直接調査を担わない
- フェーズA・B内部での孫サブエージェント委譲は許可する。
  フェーズA・Bは`agent-toolkit:plan-mode`スキルを呼び出してその規定に従う。
  例えばフェーズA内の調査委譲先で`Explore`を呼ぶ。
  フェーズB内の工程7では`plan-reviewer`とcodexレビューを並列起動する
- ユーザー対話（`AskUserQuestion`）はメインエージェント側で実施する。
  サブエージェントから直接ユーザーへ質問できないため、フェーズA完了報告の確認事項リストを経由する
- フェーズB完了報告で追加の未確定事項が判明した場合は
  メイン側で再度`AskUserQuestion`を発行し、新規サブエージェント起動で反映する

## 5. コミット

全テーマの計画ファイル作成完了後、`agent-toolkit:commit`スキルを呼び出して単一コミットでまとめて反映する。
個々のテーマ計画ファイル作成中はコミットしない。

コミット件名案: `docs(plans): v{next}計画ファイル{count}件を作成する`

## 6. 次ステップ誘導

全テーマの計画ファイル作成完了後、以下のプロンプト例をユーザーへ提示して本スキルを終了する。

> `agent-toolkit:spec-driven-impl`スキルを起動してください。
