# lint設定緩和の判断基準

textlint・markdownlint・pyfltrの推奨設定例などで、検出ルールの緩和・無効化・除外指定の追加を検討する際に用いる判断基準。
意味のある検出を完全無効化することを避けつつ、誤検知・過剰検知への対処方針を確定する。

## 根本目的（緩和判断の照合基準）

lint設定全体は次の目的に収束する。
緩和判断は当該検出ルールがいずれかの目的を支えているかで評価する。

- LLM風の癖を排除したフォーマルな日本語表現の維持
  - 口語表現・比喩動詞・主観評価語・空虚な形容・接続の型などの混入を防ぐ
  - 該当ルール例: `preset-ja-technical-writing`配下のルール、`scripts/check_colloquial.py`
- Markdownとしてセマンティックに正しい記述の維持
  - 強調・コードブロック・見出し階層の意味的役割を保つ
  - 該当ルール例: markdownlintの`MD022`・`MD031`・`MD040`等
- 折り返し無しのテキストエディターで見やすい状態の維持（127幅以内・1文1行）
  - 該当ルール例: markdownlintの`line-length`、`scripts/check_line_width.py`

## 緩和判断のフレーム

検出ルールが上記目的のどれも支えていないと判断できる場合のみ緩和を検討する。

1. 検出された違反の意味解析: 当該ルールが何を防ぐためにあるかを公式ドキュメント・`references/textlint-violations.md`で確認する
2. 根本目的との照合: 検出がいずれかの根本目的を支えているかを確認する。支えている場合は緩和せず、文面側で対処する
3. 仕様上の不可避性確認: ルールが特定構文（コード識別子の`.`を句点へ自動変換するなど）で仕様上不可避な誤変換を起こす場合のみ無効化を採用する
4. 影響範囲の限定: 緩和を採用する場合、最小範囲（特定ファイルの`.textlintignore`登録・特定行の`<!-- textlint-disable -->`コメント）を優先し、グローバル無効化は不可避性が明確な場合のみとする

## 緩和手段の選択順序

緩和が必要と判定した場合、次の順序で手段を選ぶ。

1. 文面側の表現変更で違反を回避する（推奨）
2. 該当ファイル単位の除外指定（`.textlintignore`等）
3. 該当行単位の無視コメント（理由を併記する）
4. 該当ルールの設定パラメーター調整（厳密度の緩和）
5. 該当ルールのグローバル無効化（仕様上不可避な場合のみ）

## pyfltr推奨設定例への反映

pyfltr配布の`docs/guide/recommended.md`・`docs/guide/recommended-nonpython.md`へ反映する場合、上記判断フレームに従って既存推奨設定を見直す。
緩和を採用する箇所はその根拠（根本目的のいずれも支えていない・仕様上不可避など）を設定ファイル側のコメントとして残す。

## 検証用サブコマンド

緩和の妥当性を計画段階で検証するため、対象lintツールの検証用サブコマンド（`textlint --rule <name>`・`markdownlint --rules <name>`等）を実機実行して、緩和前後の検出件数差を確認する。
