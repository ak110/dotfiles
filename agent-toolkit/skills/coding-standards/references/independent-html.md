# 単体HTML成果物のスタイル規範

単体成果物としてのHTMLは、ブラウザで開くだけで表示完結する用途を想定するため、
Webシステム内蔵HTMLとは異なるスタイル規範を要する。本referenceは当該用途向けの実装指針を集約する。

## 適用範囲

対象はユーザーへ単体成果物として提示するHTML（レポート・ダッシュボード・可視化・プレゼン・プロトタイプ等）とする。
Webシステムの一部として動作するHTML（既存フレームワークのテンプレート・コンポーネント等）は対象外とし、
当該フレームワーク・プロジェクトの規範に従う。

## 用途別の代表パターン

- 仕様・計画・探索: 複数デザイン案の並置比較、実装計画の可視化、データフロー図
- コード理解・レビュー: 差分表示と注釈、フローチャート、モジュール構造の説明
- デザイン・プロトタイプ: インタラクション試作、スライダーでの値調整、アニメーション検証
- レポート・研究: 情報統合、SVG図表、スライドショー形式
- カスタム編集UI: 優先度付け、フラグ設定管理、プロンプト調整

## 実装指針

- 自己完結性を重視し、外部ファイルへ分割せず単一ファイルで完結させる
- スタイルはCDN経由のCSSフレームワーク（`daisyUI`等）で統一し、インラインスタイルの多用を避ける
- テーマは`data-theme`属性等のフレームワーク機能で切り替え可能にする
- 情報密度を高めるため、表・SVG図表・スクリプトを1画面へ統合する
- 大量情報を扱う場合は、ナビゲーション構造（アンカーリンク・目次）で可読性を確保する
- エクスポート機能（JSON・Markdown等でコピー可能なボタン）を検討する
- リアルタイム更新が有効な用途では、左側にテンプレート編集・右側にプレビューの並置構成を検討する

## 技術選択の指針

標準のHTML/CSS/JavaScriptで成立する範囲は独自実装せず、フレームワークをCDN経由で読み込む。
外部APIやサーバー連携を要する構成は単体で完結する成果物の用途から外れるため、Webシステム側の規範に従う。
