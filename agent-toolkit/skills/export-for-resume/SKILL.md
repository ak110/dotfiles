# 次セッションへの引き継ぎ用markdown出力

会話が発散してコンテキストが肥大化した場合、または要件確認と手戻りが重なって進捗把握が困難になった場合に、
現状の要点を整理したmarkdownを作業プロジェクトルート直下に出力する。
出力ファイルを添えて新しいClaude Codeセッションを起動し、冒頭でそのファイルを読み込むことで、
これまでの経緯を短く伝えて作業を再開できる。

## 用途と前提

- 想定場面:
  - 要件対話や計画ファイル作成中に議論が発散し、何が決まって何が未決かの把握が困難になった場合
  - コンテキストが肥大化して応答品質や速度が低下してきた場合
  - セッション終了直前に、現時点のスナップショットだけ残して後日再開したい場合
- 前提: ユーザーが明示的にこのスキルを呼び出している状態。実行中のコード変更やコミット操作は行わず、ファイル出力のみを実行する

## 出力先とファイル名

- 出力先: 作業プロジェクトのルート直下
- ファイル名: `resume-<会話テーマを要約した英語文>.md`
 （会話の主題を短い英語のケバブケースで要約する。例: `resume-agent-toolkit-cleanup.md`）
- ユーザーがgit管理外として扱うか任意の場所へ移動するかはユーザー判断に委ねる

同一テーマで再呼び出しされた場合は既存ファイルの上書きでよい
（更新日時はファイルシステム側で追跡できるため本文にタイムスタンプは含めない）。

## 出力テンプレート

以下のテンプレートに沿ってmarkdownを生成する。
情報がない節は「（該当なし）」と明記して空節にしない。

```markdown
# セッション引き継ぎメモ

## 発端の指示

<ユーザーの初期要望・最初のトリガーとなった依頼内容を原文ベースで>

## 決定事項

- <ユーザーとの対話で合意済みの方針・仕様・制約>
- ...

## 未決事項

- <合意に至っていない論点・ユーザー確認待ちの質問>
- ...

## 現在の進捗

- <着手済みタスクとその状態>
- <未着手タスク>

## 関連ファイル・パス

- 計画ファイル: <`~/.claude/plans/*.md`のパスがあれば>
- 編集対象ファイル: <絶対パスまたはリポジトリ相対パス>
- 参考ドキュメント: <参照したドキュメントのパス>

## これまでの試行錯誤の要点

- <検討したが採用しなかった方針・棄却理由の要約>
- <問題が発生した箇所と解消経緯>
```

## 出力後のユーザー案内

ファイル生成後、以下の要点を口頭でユーザーに伝える（ファイル内には書かない）。

- 出力ファイルの絶対パス
- 新しいClaude Codeセッションを起動し、冒頭でこのファイルを読み込んでから作業を再開する運用を推奨する旨
- 本ファイルはgit管理外として扱うか、任意の場所へ移動するかをユーザーが判断できる旨

## 注意点

- 決定事項と未決事項を厳密に分ける。推測や前提で未決事項を埋めない（次セッションでの誤解を招くため）
- ユーザーの発話・確認済み合意のみを「決定事項」に書く。エージェントの推測は書かないか、明示的に「推測」と注記する
- 過去の試行錯誤も簡潔に残す。同じ論点を再検討して同じ結論に至る手戻りを防ぐため
- コード断片や詳細仕様は極力含めず、参照先のファイルパスで代替する。引き継ぎの目的はコンテキスト削減のため
- 本スキル実行中に推測判断が必要な箇所があれば、先にユーザーに確認して確定させる（スキル本体の趣旨に反するため）
