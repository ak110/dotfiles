# 実装差分レビュー初回プロンプト

`plan-codex-delegate`（用途: 実装差分レビュー）の初回プロンプトテンプレート。
`{plan_impl_reviewer_agent_path}`・`{review_standards_skill_path}`は
`plan-codex-delegate`が本文中の`${CLAUDE_PLUGIN_ROOT}`基準パスで埋め込む。
`danger-full-access`サンドボックスのcodexはプロジェクトcwd外の絶対パスも読み込み可能である。
プロンプト共通化の設計方針は、claude側`plan-impl-reviewer.md`と定義を共有する。

以下がプロンプト本文である。

```text
以下の実装についてレビューを実施して。プロジェクトルート: {project_directory}

- 対象範囲: {ファイルパス列挙またはgit範囲}
- 対象範囲スコープ: 既存ファイル全体（既存違反も指摘してよい）。計画ファイルで別途明示があればそちらに従う
- 対象外ファイル: {対象外ファイル一覧}
- 計画ファイル: {計画ファイルパス}（計画ファイル無しの場合は本欄を省略）
- 差分取得: {差分取得コマンド}
- 担当観点: {担当カテゴリ（コード・テストコード／一般ドキュメント／コーディングエージェント向け文書）}に
  該当する項目のみを対象とする。
  観点定義は`{plan_impl_reviewer_agent_path}`「担当観点」節を直接読み込んで適用すること
- レビューの基本姿勢・重大度ラベル・全件報告・構文合法性の除外・差分起因と既存違反の区分は
  `{review_standards_skill_path}`を直接読み込んで適用すること
```
