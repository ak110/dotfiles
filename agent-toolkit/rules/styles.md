# styles.md

コード・テストコード・ドキュメントなどあらゆる成果物の新規作成・修正・計画・レビュー時に、以下の方針に従う。

## 大原則

- 自然で簡潔な日本語で書く

## 対象読者と文体

成果物の対象読者を考慮し、適切な文体・表現を使用する。

| 読み手 | 文書／出力の例 | 文体 |
| --- | --- | --- |
| エンドユーザー | 対話型UI（CLI案内・GUIダイアログ・エラー画面・通知） | NHKスタイル |
| エンドユーザー | README・利用ガイド・`docs/guide/`配下 | JIS規格スタイル |
| 開発者 | アーキテクチャ・CONTRIBUTING | JIS規格スタイル |
| 開発者 | コーディングエージェントの発話（応答本文および拡張思考） | JIS規格スタイル |
| コーディングエージェント | コーディングエージェント向け文書[^1]・コードコメント | JIS規格スタイル |

[^1]: `CLAUDE.md`・`.claude/rules/`・`SKILL.md`・サブエージェント定義など、コーディングエージェントが直接参照する文書

### 文体：JIS規格スタイル

ドキュメントなど常体を使用する文章は **JIS規格・公的な標準仕様書のスタイル** の文体に統一する。
語尾・漢語率・断定度・規範述語（「〜とする」「〜ものとする」「〜してはならない」「〜できる」）をそのまま踏襲する。

### 文体：NHKスタイル

対話型UIなど敬体を使用する文章は **NHKの案内放送原稿のスタイル** の文体に統一する。
明瞭で中立的な表現を用いる。命令調、断定的な責め口調、過度に事務的な表現を避ける。

### コーディングエージェントの発話

進捗・選択判断・成果物状況の情報量を保ち、拡張思考・ツール呼び出し前後の短文ステータス報告も日本語で書く。
結論ファーストで構成し、挨拶・前置き・絵文字を含めず、指摘すべき事項は曖昧化せず率直に記述する。
コード識別子・英語キーワードに引きずられて文全体を英語にしない。

### 個別注意

- 複数ファイル横断の修正・計画ではファイルごとに対象読者・文体・粒度が変わることに注意し、他ファイルの文体をそのまま適用しない
- コメント・ログ・ドキュメント・コミットメッセージなど全ての文章を日本語で書く（プロジェクト方針が無い場合）
- 対話型UI向けでも、コーディングエージェントに直接渡るhook出力（`reason`・`additionalContext`・exit 2のstderr等）は
  プロジェクト方針で英語表記を固定する場合がある。
  Claude Code関連の出力先別方針は`agent-toolkit:agent-standards`スキルの
  `references/claude-hooks.md`「メッセージの記述言語」節に従う
